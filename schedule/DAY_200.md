## Block 1 — The one idea: methodology over models
Every day of this module served one thesis: **in finance, how you do ML matters far more than which model you use.** The model is a one-line commodity; the craft is everything around it:
- **respect the domain** — low signal, non-stationary, tiny samples, overfit-prone (191),
- **engineer features & labels honestly** — frac-diff, triple-barrier, uniqueness weights (192),
- **validate without leakage** — purging, embargo, CPCV (193),
- **regularize hard, prefer simple** — trees over nets unless data structure demands otherwise (194, 196),
- **understand the model** — MDA/SHAP, detect leakage and noise-fitting (195),
- **find new data** — alt-data/NLP where the edge has migrated (197),
- **use RL for control, not prediction** (198),
- **judge whether the edge is real** — deflated Sharpe, PBO (199).

Keep one thing from Phase 12: **ML in finance is 80% not-fooling-yourself and 20% modeling.** Whoever validates hardest wins.

## Block 2 — The honest ML pipeline (your mental flowchart)
```
1. Sanity prior: signal is faint, samples tiny, overfitting is the default. Stay humble.
2. Features: stationary-with-memory (frac-diff), lookahead-safe, economically motivated.
3. Labels: triple-barrier (vol-scaled); meta-label for sizing; uniqueness sample weights.
4. Model: simple & regularized first (linear → GBM); DL only if data structure demands it.
5. Validate: purged + embargoed CV / CPCV — NEVER naive k-fold.
6. Interpret: MDA/SHAP — confirm concentrated, economically-sensible importance; hunt leakage.
7. Judge: count trials → Deflated Sharpe; compute PBO; truly-OOS test; economic story.
8. Reality: net of costs & capacity (Day 179); robust across regimes; not crowded.
9. Decide: deploy only if it survives ALL of the above — else walk away.
```
This routes any ML-driven strategy from idea to (rarely) deployment.

## Block 4 — Mathematics · the relationships worth knowing cold
```
Fractional diff:   minimum d making the series stationary while keeping memory
Triple-barrier:    label = first of {profit-take, stop-loss, time-limit} hit
Purged CV:         drop train obs whose label window overlaps the test set (+ embargo)
Ensemble variance: bagging averages decorrelated overfits → variance ↓ (∝ 1/√trees)
Selection bias:    E[max Sharpe of N trials] ≈ √(2 ln N) × σ_SR
Deflated Sharpe:   P(true SR>0 | observed SR, N trials, skew, kurtosis, T)
```
If you can explain why k-fold leaks, why labeling beats modeling, and why `√(2 ln N)` dooms naive backtests, you can reconstruct the module under pressure.

## Block 5 — Quant Thinking · interview rehearsal (say these out loud)
1. *Can ML beat the market?* → a useful tool for nonlinearity/alt-data, but finance is uniquely hostile (low signal, non-stationary, tiny samples); success is methodology, not horsepower (191).
2. *Where do you start on an ML strategy?* → features & labels (frac-diff, triple-barrier, uniqueness weights), not the model (192).
3. *How do you cross-validate?* → never naive k-fold; purge + embargo / CPCV; it leaks otherwise (193).
4. *Which model for tabular finance?* → regularized tree ensemble (RF/GBM); DL only for sequence/text/image with enough data (194, 196).
5. *How do you know what the model learned?* → MDA (not MDI) + SHAP; concentrated, economic importance; hunt leakage (195).
6. *Where's alpha today?* → alternative data + processing; it's migrated from modeling to data (197).
7. *RL for trading?* → control sub-problems (execution/hedging) with good simulators, not end-to-end alpha (198).
8. *Backtest looks great — real?* → count trials, deflate the Sharpe, compute PBO, test truly-OOS, demand an economic story (199).

**Hard puzzle (capstone):** you're handed a Jupyter notebook: a gradient-boosted model on 80 features predicting triple-barrier labels, with a reported Sharpe of 2.8 and "5-fold CV accuracy of 71%." You have one hour before a go/no-go meeting. What do you check, in priority order? (Triage by *most-likely-fatal* first: (1) **CV method — is it purged/embargoed?** "5-fold CV" on overlapping triple-barrier labels with autocorrelated data is **almost certainly leaking** (Day 193) — re-run with purging + embargo; the 71% will likely crater toward ~52%. This is the single most probable killer. (2) **71% accuracy is implausible** for a near-efficient market (Day 191) — that scream says **leakage**, either in CV *or* in a feature (check for any feature using future-relative info, like same-day VWAP — Day 195's leakage hunt via MDA). (3) **Trial count → Deflated Sharpe** (Day 199): how many feature sets/hyperparameters were tried to land on these 80 features and this 2.8? Without the trial count the Sharpe is uninterpretable; deflate it. (4) **Truly out-of-sample**: is there untouched data, or was everything used in the search? (5) **Feature importance (MDA, not MDI)**: concentrated & economic (trust) or diffuse (noise-fitting) or one dominant suspicious feature (leakage)? (6) **Labels**: are the triple-barrier labels vol-scaled and uniqueness-weighted, or are overlapping labels inflating performance? (7) **Costs/capacity** (Day 179): does 2.8 survive realistic transaction costs at size? **Recommendation logic:** if the CV isn't purged and accuracy is 71%, I walk into the meeting recommending **no-go pending re-validation** — the result is presumptively a leakage artifact until the purged-CV, deflated-Sharpe, OOS, and importance checks say otherwise. This one notebook exercises the entire module — and the *correct* answer is disciplined skepticism, not enthusiasm.)

**Career connection:** Phase 12 *is* the quant-ML interview, and its lesson is counterintuitive: the impressive thing isn't knowing the models (everyone does) — it's the **methodological discipline** to use them honestly. A candidate who leads with purged CV, triple-barrier labels, deflated Sharpe, MDA, and "I'd be skeptical of that backtest" demonstrates they understand the *real* job: extracting faint, decaying signal from hostile data *without fooling themselves*. That's worth more than any architecture.

## Block 6 — Python (quant) · The honest pipeline, end to end
```python
import numpy as np
from sklearn.ensemble import GradientBoostingClassifier
from sklearn.inspection import permutation_importance
from scipy.stats import norm
rng = np.random.default_rng(0)

# 1. data with a weak-but-real signal in 2 of 30 features (the realistic regime)
T,F = 2500, 30
X = rng.standard_normal((T,F))
y = ((0.35*X[:,0] + 0.3*X[:,1]*X[:,2] + rng.standard_normal(T)*1.8) > 0).astype(int)

# 2. PURGED walk-forward validation (no leakage) + collect OOS predictions
def purged_eval(model, embargo=25, n=6):
    fold=len(X)//n; scores=[]
    for i in range(1,n):
        tr=slice(0,i*fold-embargo); te=slice(i*fold,(i+1)*fold)
        model.fit(X[tr],y[tr]); scores.append(model.score(X[te],y[te]))
    return np.mean(scores)

model = GradientBoostingClassifier(n_estimators=150, learning_rate=0.02, max_depth=2, subsample=0.7, random_state=0)
acc = purged_eval(model)
print(f"purged OOS accuracy : {acc:.3f}  (honest — modest, as it should be)")

# 3. interpret (MDA), 4. deflate for trials
model.fit(X[:1800],y[:1800])
mda = permutation_importance(model, X[1800:], y[1800:], n_repeats=10, random_state=0).importances_mean
print(f"top-2 MDA features  : {np.argsort(mda)[-2:]}  (should be the real signal: 0,1,2)")
def deflated_sharpe(SR, N, T=2520):
    s=1/np.sqrt(T); SR0=np.sqrt(2*np.log(max(N,2)))*s
    return norm.cdf((SR/np.sqrt(252)-SR0)/s)
print(f"Sharpe 1.5 if I tried 50 configs -> DSR = {deflated_sharpe(1.5,50):.2f}")
```
The complete honest pipeline: a regularized model, **purged** validation (modest, believable accuracy), **MDA** to confirm the real features drive it, and **deflation** for the search. This — not the model — is the deliverable.

**Linux:** `python honest_ml.py | tee honest.log`; this template (purged CV + MDA + DSR) is the scaffold for *every* ML strategy you'll build.
