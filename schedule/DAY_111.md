## Why today matters
Students learn DP as tables/recurrences/memoization; experts see *states → transitions → graph → optimization.* **Every DP problem creates a graph.**

---

## Block 1 — Fibonacci as a graph
`f(n)=f(n-1)+f(n-2)`: states `0..n`, transitions `n→n-1`, `n→n-2`. Always moves downward → no cycles → a **DAG.**

*Why it matters:* once you see the DAG, DP's "fill in dependency order" is just topological processing — the table is incidental.

---

## Block 2 — DSA · DP = DAG processing
`A → B → C`: to compute `dp[C]` you need `dp[B]`, which needs `dp[A]` — identical to **topological processing** of a DAG.

**Deep insight:** DP works because *dependencies form a DAG*, not because of tables.

---

## Block 3 — DSA · Grid DP as graph
Grid state `(row, col)`, transitions right/down → a cell points to neighbour cells, movement never goes backward → a **DAG.** Unique Paths = "number of ways to reach a node."

**Signal:** whenever you see *state + transitions*, ask *can I draw a graph?*

---

## Block 4 — DSA · Longest Path in a DAG
`1→2→4`, `1→3→4`: `dp[node]` = best value ending here; `dp[v] = max(dp[v], dp[u] + reward)`. This is graph **and** DP at once.

**Huge insight:** Graph DP isn't "Graph + DP" — it's *DP viewed explicitly as a graph.*

---

## Block 5 — Mathematics (Optimization over states)
State graph `A→B=5, A→C=2, C→B=1`: shortest A→B = 3. DP says `dp[B] = min(through A, through C)` — same idea, **choose the best parent.**

**Exercise:** `A→C, B→C`, `dp[A]=10, dp[B]=20`, maximising → which parent? **Challenge:** why does a cycle break many DP formulations? (No valid dependency order.)

---

## Block 6 — Quant Thinking (Research as a DAG)

`Probability → Statistics → ML → Quant Research` — each topic creates a future state, studying is a transition; your roadmap is a **state graph.** "Which path creates maximum future value?" = optimization on a DAG.

**Hard puzzle:** path Prob→Stats→ML (value 100) vs Prob→Optimization→Quant Research (value 120) — how would DP choose?

**Career connection:** portfolio allocation, research planning, feature engineering, signal selection are all *states → decisions → new states* — strong modelers see the graph inside the optimization.
