## The story
Stalls at positions `1 2 4 8 9`, place **3 cows** so the **minimum distance between any two cows is as large as possible.** Placing at `1, 4, 8` gives min distance 3 — and that's the maximum achievable.

---

## Block 1 — C++ (Understand the objective)
We're not asked *can we place cows?* (easy) — but *what is the **largest** minimum distance?* That's an optimization problem.

*Why it matters:* "maximise the minimum" / "minimise the maximum" phrasings are the fingerprint of binary-search-on-answer.

**Code from scratch:** `aggressive_cows.cpp`; store stalls; **sort first, always.**

**Thinking question:** why does sorting help? (Greedy placement needs ordered positions.)

---

## Block 2 — DSA (Transform optimization → decision)
Instead of *largest distance?*, ask *can cows be placed with minimum distance D?* Define `check(D)` → true/false. For `D=3` on `1 2 4 8 9`: place at 1, then 4, then 8 → 3 cows placed → **true.**

---

## Block 3 — DSA (Greedy feasibility check — the key)
To test `check(D)`: place the first cow at the first stall, then **always place the next cow at the earliest stall ≥ last + D.** For `D=4`: place 1, then 8 → only 2 cows → need 3 → **false.**

---

## Block 4 — DSA (Monotonic property)
```
check(1)=T  check(2)=T  check(3)=T  check(4)=F  check(5)=F  → TTT FF
```
A clean boundary → binary search works. **Signal:** small answer works, large fails (or vice versa) → look for monotonicity.

---

## Block 5 — Mathematics (Answer space)
Stalls `1..9`: min distance 1, max distance `9−1 = 8`. Search range `[1, 8]`, binary search `O(log R)`, each check `O(n)` → total **O(n log R)**.

**Exercise:** stalls `1 5 10 20` — what's the answer range? (`[1, 19]`.)

---

## Block 6 — Quant Thinking (Feasibility before optimization)

We never solved "maximum distance" directly — we solved "can distance D work?" then searched. Same shape everywhere: *"can the system handle 1000 req/s?"* → check → search capacity; *"can skill level X be reached in 6 months?"* → check → search.

**Hard puzzle:** the highest DSA difficulty you can solve consistently — how would you define `check(level)`?

**Career connection:** resource allocation, capacity planning, scheduling, trading infra, network design — many systems use *binary search + feasibility* instead of optimizing directly.
