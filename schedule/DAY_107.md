## Block 2 — DSA · DAG + DP Framework
For `1→2→4`, `1→3→4` with rewards `1=5, 2=10, 3=20, 4=30`:
```
dp[node] = max reward of a path ending at node
init dp[node] = reward[node]
for edge u→v:  dp[v] = max(dp[v], dp[u] + reward[v])
```

---

## Block 3 — DSA · Why topological sort matters
Before computing `dp[4]` you need `dp[2]` and `dp[3]` — so process **parents first** in topological order (`1 2 3 4` or `1 3 2 4`).

**Signal:** dependencies / prerequisites / workflow / pipeline → DAG; add optimization → Graph DP.

## Block 5 — Mathematics (State dependency)
Fibonacci depends on previous numbers; Grid DP on previous cells; **Graph DP on parent nodes.** General form: `dp[state] = best(previous states)`.
