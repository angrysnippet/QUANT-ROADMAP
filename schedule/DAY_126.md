## Block 1 — Power = 1 − β
**Power** is the probability of detecting an effect that's truly there. It rises with: bigger **effect size**, larger **sample size `n`**, and lower noise `σ`. Low power → you'll *miss* real edges (and the ones you do "find" are exaggerated — the **winner's curse**).

*Why it matters:* a backtest that can't statistically detect your edge even if it's real is worthless — and most short backtests are exactly that.

---

## Block 2 — The sample-size relationship
To detect an effect of size `δ` against noise `σ` at significance α with power `1−β`:
```
n  ∝  (σ / δ)²            (roughly  n ≈ (z_α + z_β)² · σ² / δ²)
```
**Small effect → quadratically more data.** Halving the detectable edge needs ~4× the sample. This is the √n law wearing a different hat.

---

## Block 3 — Translating to "years to detect a Sharpe"
Since `t ≈ Sharpe·√years`, to reach `t = 2` (≈ significant):
```
years ≈ (2 / Sharpe)²
```
- Sharpe 2.0 → ~1 year. Sharpe 1.0 → ~4 years. Sharpe 0.5 → ~16 years.

**Exercise:** a strategy you believe has Sharpe 0.7 — roughly how long to confirm it? (`(2/0.7)² ≈ 8 years`.) Lesson: **low-Sharpe edges are nearly unverifiable** with limited history.

---

## Block 4 — Mathematics (the winner's curse)
When power is low, the *estimated* effect of any "significant" result is biased **upward** (you only cross the threshold when noise helps). So an underpowered backtest's Sharpe is optimistic even when the edge is real.

**Challenge:** why does combining "low power + multiple testing" (Day 125) produce backtests that are both fake *and* inflated? (You select the luckiest of many noisy, underpowered estimates.)

---

## Block 5 — Quant Thinking (Power discipline)

- **Pre-compute required `n`** before trusting a result: can this dataset *possibly* detect the edge I'm claiming?
- **High-frequency strategies** get power from millions of trades (huge `n`) — they can confirm tiny edges. **Low-frequency macro bets** have tiny `n` (few independent events) and are nearly impossible to validate statistically — judgment matters more there.
- **Effective sample size** ≪ data points when observations are autocorrelated (regimes) — discount accordingly.

**Hard puzzle:** two PMs: one trades intraday (5,000 indep. trades/yr, Sharpe 0.5), one trades a macro thesis (~6 indep. bets/yr, Sharpe 1.5). Whose track record can you statistically trust sooner? (The intraday PM — vastly more independent observations; the macro PM may *never* be statistically distinguishable from luck within a career.)

**Career connection:** "how much data do you need to validate this strategy?" → answer with `years ≈ (2/Sharpe)²` and a hard look at independence. Senior-level instantly.

## Block 6 — Python (quant) · Power by simulation
```python
import numpy as np
rng = np.random.default_rng(0)
def power(sharpe_daily, n, trials=5000, alpha=0.05):
    z = 1.96
    hits = 0
    for _ in range(trials):
        r = rng.normal(sharpe_daily, 1.0, n)     # daily returns, unit vol
        t = r.mean()/(r.std(ddof=1)/np.sqrt(n))
        if t > z: hits += 1
    return hits/trials
print(power(0.05, 252))   # ~1yr, low-Sharpe edge -> low power
print(power(0.05, 252*8)) # 8yrs -> high power
```
