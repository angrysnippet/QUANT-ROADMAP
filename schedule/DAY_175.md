## Block 1 — The problem: relationships drift
The hedge ratio between two stocks isn't constant — it changes as their businesses, leverage, and factor exposures evolve. A **fixed β** (Day 172) goes stale; a **rolling-window β** (Day 174) adapts but is clumsy: it lags, weights all points in the window equally then drops them abruptly (the "ghost" effect, Day 143), and forces you to pick a window length. We want a `β` that **updates smoothly and optimally every day** as new data arrives. That's exactly what the Kalman filter does.

*Why it matters:* a better, adaptive hedge ratio = a tighter, more stationary spread = a cleaner signal and fewer false breakdowns.

---

## Block 2 — State-space thinking
Model the world as a hidden **state** that evolves, observed only through noisy measurements:
- **State (hidden):** the true hedge ratio `β_t` (and maybe an intercept), which drifts slowly:  `β_t = β_{t−1} + w_t` (a random walk).
- **Observation (noisy):** `A_t = β_t · B_t + v_t` — you see the prices, not `β_t` directly.

This is a **state-space model** (Day 109's state thinking, now stochastic). The Kalman filter is the *optimal* estimator of the hidden state given the noisy observations — it's the recursive, real-time engine for "what's the true β right now?"

---

## Block 3 — Predict and update (the two-step heartbeat)
The Kalman filter cycles two steps each period — pure **Bayesian updating** (Day 122) in disguise:
1. **Predict:** project the state forward (`β` stays, uncertainty grows by the process noise).
2. **Update:** a new observation arrives; correct the prediction in proportion to the **Kalman gain** `K`:
```
β_new = β_predicted + K · (observed − predicted)
```
The **Kalman gain** optimally balances *how much you trust the model* (prior) vs *how much you trust the new data* (evidence): high measurement noise → small `K` (ignore the noisy point); high process noise (β drifts fast) → large `K` (adapt quickly). It's the optimal Bayesian compromise, computed recursively — **no window to choose, no lag from equal weighting, and it carries its own uncertainty estimate.**

**Concept check:** measurements suddenly get very noisy (volatile, illiquid). What happens to the Kalman gain and to `β`'s updates? (Gain *shrinks* — the filter trusts its prior more and barely moves `β` on each noisy point, automatically becoming more conservative. That adaptivity is the magic.)

---

## Block 4 — Mathematics · why Kalman is *optimal*
For a **linear** system with **Gaussian** noise, the Kalman filter is provably the **minimum-mean-squared-error** estimator of the state — and equivalently the exact recursive Bayesian posterior (the posterior stays Gaussian, so it's fully described by a mean and a covariance that the filter propagates). It's also recognizable as **recursive least squares** with a forgetting factor — exponentially discounting old data in a principled, automatic way (compare EWMA, Day 143, but optimal and self-tuning). Two parameters control its behavior:
- **process-noise variance `Q`** — how fast you let `β` drift (bigger Q = more adaptive, noisier),
- **measurement-noise variance `R`** — how noisy you think the prices are (bigger R = smoother, slower).

The ratio `Q/R` is the one real knob: it sets the adapt-vs-smooth trade-off — the **bias–variance trade-off** (Day 128) wearing yet another costume. (For nonlinear systems you extend to the EKF/UKF or particle filters — same predict–update skeleton.)

## Block 5 — Quant Thinking · one filter, everywhere
The Kalman filter is a *general-purpose adaptive estimator*, and once you have it you see uses across the whole quant landscape:
- **dynamic hedge ratios** for pairs/baskets (today),
- **time-varying beta** to the market or factors (Day 148) — betas drift, Kalman tracks them,
- **trend/level extraction** from noisy prices (a smarter moving average),
- **stochastic-volatility and term-structure filtering** (estimating hidden vol/rate states, Days 143/168),
- **sensor fusion / signal de-noising** far beyond finance (it landed Apollo on the Moon).

The unifying idea — **maintain a belief about a hidden state, predict it forward, and update it optimally as noisy evidence arrives** — is one of the most transferable algorithms you'll ever learn. It's Bayesian updating made recursive and real-time, and it appears the moment you have *a hidden quantity observed through noise*, which is… most of quant finance.

**Hard puzzle:** a colleague uses a 60-day rolling β and complains it's "always either too laggy or too jumpy depending on the window." How does the Kalman filter dissolve this dilemma — and does it *remove* the trade-off? (The rolling window forces a single hard choice — short window = jumpy/noisy, long = laggy/stale — and weights every point equally then discards it abruptly. Kalman replaces the hard window with a *smooth, optimal* exponential weighting driven by the `Q/R` ratio, and crucially **adapts the effective memory to conditions** (noisy data → trust prior; clean data → adapt fast), while tracking its own uncertainty. But it does **not** abolish the trade-off — `Q/R` *is* the bias–variance dial (Day 128); you've replaced "pick a window" with "pick `Q/R`," just in a more principled, self-adjusting, uncertainty-aware form. There's no free lunch — only a better-shaped one.)

**Career connection:** "your hedge ratio keeps drifting — how do you handle it?" Strong answer: model β as a hidden state in a **state-space model** and estimate it with a **Kalman filter**, which updates β optimally each day (predict–update / Bayesian), trades off responsiveness vs smoothness via the process/measurement-noise ratio, and carries its own uncertainty — far cleaner than a fixed or rolling window. Bonus: note it's recursive least squares / real-time Bayes, and that the one tuning knob is the bias–variance trade-off. That answer signals serious tooling depth.

## Block 6 — Python (quant) · A Kalman-filtered dynamic hedge ratio
```python
import numpy as np, pandas as pd
rng = np.random.default_rng(0)

# simulate a pair whose TRUE hedge ratio drifts over time
T=1500; B=np.cumsum(rng.standard_normal(T))*0.5 + 50
beta_true = 1.0 + 0.3*np.sin(np.arange(T)/300)            # slowly drifting β
A = beta_true*B + rng.standard_normal(T)*2.0

# 1-D Kalman filter for the hedge ratio  (state = beta_t)
Q = 1e-4      # process noise: how fast we let beta drift
R = 4.0       # measurement noise: how noisy prices are
beta, P = 0.0, 1.0
est = np.zeros(T)
for t in range(T):
    # predict
    P = P + Q
    # update (observation A_t = beta * B_t + noise)
    K = P*B[t] / (B[t]**2 * P + R)               # Kalman gain
    beta = beta + K*(A[t] - beta*B[t])
    P = (1 - K*B[t])*P
    est[t] = beta

print("final estimated β:", round(est[-1],3), " true:", round(beta_true[-1],3))
print("tracking error (last 500):", round(np.mean(np.abs(est[-500:]-beta_true[-500:])),3))
# the Kalman β tracks the drift; a fixed OLS β would sit at the average and mis-hedge
ols = np.polyfit(B, A, 1)[0]
print("static OLS β:", round(ols,3), " (misses the drift entirely)")
```
The Kalman estimate *tracks the moving true β*, while a single OLS β sits at the average and systematically mis-hedges — a dynamic hedge ratio that learns, in ~10 lines.

**Linux:** `python kalman.py | tee kalman.log`; raise `Q` and watch β adapt faster but noisier — the `Q/R` bias–variance dial, live.
