## Block 1 — The new way to look at problems
Most hard problems have four layers: **optimization → decision version → efficient check → algorithm.** "Find minimum capacity" → "can capacity X work?" → greedy check → binary search.

*Why it matters:* peeling these layers is the repeatable method for problems that look impossible at first glance.

---

## Block 2 — DSA · Binary Search + Greedy Framework
1. Define the answer (e.g. capacity).
2. Define `check(capacity)`.
3. Use **greedy** inside the check (load packages until the load exceeds capacity → start a new day).
4. Return true/false.
5. Binary search the answer.

---

## Block 3 — DSA · Pattern Recognition
- **Aggressive Cows:** answer = distance; check = can cows be placed?
- **Allocate Books:** answer = max pages; check = can books be assigned?
- **Ship Packages:** answer = capacity; check = can packages ship in D days?

**Deep insight:** all three are secretly **find-the-boundary** problems.

## Block 4 — C++ · Framework Template
Create `binary_search_greedy.cpp`:
```cpp
bool check(long long mid) { /* greedy simulation */ }
while (low <= high) {
    long long mid = low + (high - low) / 2;
    if (check(mid)) { ans = mid; high = mid - 1; }
    else low = mid + 1;
}
```
Memorise the *structure*, not the problem.

---

## Block 5 — Mathematics (Boundary search)
`F F F T T T` → find the **first True.** Binary search specialises in boundary detection.

**Exercise:** for `T T T T F F F`, which boundary? **Challenge:** why can't binary search work on `T F T F T`? (No single boundary — not monotonic.)

---

## Block 6 — Quant Thinking (Engineering through feasibility)

Real systems ask *"can this support 100,000 users?"* — if yes, try 200,000; if no, reduce. That's binary search + feasibility.

**Examples:** cloud infra (min servers? can N handle load?); learning roadmap (min study hours/day? can the target be hit?).

**Career connection:** portfolio limits, risk thresholds, execution systems, capacity planning all use *test → evaluate → search* rather than closed-form formulas.
