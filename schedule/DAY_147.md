# Day 147 — 💹⚖️ Finance & Portfolio World · Modern Portfolio Theory — The Efficient Frontier

> Markowitz (1952) turned investing into **optimization**: choose weights to minimize `wᵀΣw` for a target return. The output is the **efficient frontier** and, with a risk-free asset, the single best risky portfolio — the **tangency portfolio**.

**Focus:** Mean-Variance Optimization · Efficient Frontier · Tangency / Max-Sharpe · Capital Market Line · MPT's Fragility

## Block 1 — The optimization problem
MPT formalizes Day 144's "up and to the left":
```
minimize   wᵀ Σ w           (risk)
subject to wᵀ μ = target,   Σ wᵢ = 1     (and optionally wᵢ ≥ 0)
```
For each target return there's a *minimum-variance* portfolio. Trace them all → a curve in (σ, μ) space. This is mean-variance optimization (MVO): the first place your linear algebra, calculus, and Lagrange multipliers earn their keep in finance.

*Why it matters:* it's the rigorous statement of "diversify intelligently" — not equal weight, but the weights that squeeze the most return out of each unit of risk given **Σ** and **μ**.

---

## Block 2 — The efficient frontier
Plot every minimum-variance portfolio. The upper edge — where you can't get more return without more risk — is the **efficient frontier**. Anything *below* it is dominated (same risk, less return → irrational to hold). The frontier is a hyperbola; its left tip is the **global minimum-variance portfolio** (the one point you can build *without even knowing μ* — which matters, because μ is the hard part).

**Concept check:** why is a single stock almost never on the efficient frontier? (Its idiosyncratic risk is uncompensated — a diversified combo achieves the same expected return at lower σ, Day 146.)

---

## Block 3 — Add a risk-free asset → the tangency portfolio
Introduce a risk-free rate `r_f`. Now you can mix cash with risky portfolios. The best risky portfolio is where a line from `r_f` is **tangent** to the frontier — the **tangency portfolio**, which has the **maximum Sharpe ratio** of all portfolios. Closed form:
```
w_tangency ∝ Σ⁻¹ (μ − r_f·1)
```
Every investor should hold *the same* tangency portfolio of risky assets, then slide along the straight line to it — the **Capital Market Line (CML)** — using cash or leverage to hit their personal risk level. That's the **Two-Fund Separation Theorem**: risk *preference* sets the mix of cash vs tangency; it does **not** change *which* risky portfolio is optimal.

---

## Block 4 — Why naïve MPT fails in practice (this is the real lesson)
Plug in sample `μ̂` and `Σ̂`, and the optimizer behaves like a **maximum of estimation error**:
- it's wildly sensitive to `μ` — and expected returns are nearly impossible to estimate (a tiny `μ` change → huge weight swings),
- it produces **extreme, concentrated, often huge long/short** weights,
- `Σ̂` is noisy and near-singular for many assets (Day 146) → `Σ⁻¹` amplifies the noise,
- it's **out-of-sample fragile** — frequently beaten by naïve **1/N** equal weighting (DeMiguel et al., 2009).

Michaud called MVO an "error-maximizer." The fixes are the actual craft:
- **shrinkage** of μ (toward the grand mean) and Σ (Ledoit–Wolf, Day 122),
- **weight constraints** (no shorting, position caps),
- **risk-based portfolios** that avoid μ entirely: min-variance, **risk parity**, max-diversification,
- **Black–Litterman** — blend a market-equilibrium prior with your views (Bayesian, Day 122),
- **robust / resampled** optimization.

**Hard puzzle:** your MVO hands you +340% in one stock and −280% in another. Nothing is "wrong" with the code — so what's wrong, and what do you do? (Estimation error in μ/Σ amplified by Σ⁻¹; the optimizer chases noise. Fixes: shrink μ and Σ, add no-short and position-cap constraints, or drop μ and run min-variance/risk-parity. Sanity-check against 1/N out-of-sample.)

## Block 5 — Quant Thinking · the model is right, the inputs are the problem
MPT's logic is *correct and foundational* — risk is `wᵀΣw`, the frontier and tangency portfolio are real. What's fragile is **garbage-in**: the estimates of μ and Σ. This is the recurring quant lesson — **the framework is the easy 20%; robust estimation, constraints, and validation are the hard 80%.** "Optimize the portfolio" is a one-liner; making the optimizer *survive out-of-sample* is the whole job.

**Career connection:** "would you put sample means into a mean-variance optimizer?" The senior answer is an emphatic *no*: it error-maximizes, gives crazy weights, and loses to 1/N out-of-sample — then you list shrinkage, constraints, risk-parity, and Black–Litterman. Knowing *why MPT fails* signals far more experience than reciting the frontier.

## Block 6 — Python (quant) · Efficient frontier + max-Sharpe, with constraints
```python
import numpy as np
from scipy.optimize import minimize
rng = np.random.default_rng(9)

N = 6
mu    = np.array([.08,.10,.12,.07,.11,.09])              # annual expected returns
vols  = np.array([.15,.20,.25,.12,.22,.18])
C     = 0.3*np.ones((N,N)); np.fill_diagonal(C,1.0)
Sigma = np.outer(vols,vols)*C
rf    = 0.03

def port(w): return w@mu, np.sqrt(w@Sigma@w)
def neg_sharpe(w): r,s = port(w); return -(r-rf)/s

cons = ({'type':'eq','fun':lambda w: w.sum()-1},)        # fully invested
bnds = [(0,1)]*N                                         # long-only (a key constraint!)
w0   = np.full(N,1/N)
res  = minimize(neg_sharpe, w0, bounds=bnds, constraints=cons)

r,s = port(res.x)
print("max-Sharpe weights:", res.x.round(3))
print(f"return={r:.3f}  vol={s:.3f}  Sharpe={(r-rf)/s:.2f}")

# global minimum-variance portfolio (needs no mu)
gmv = minimize(lambda w: w@Sigma@w, w0, bounds=bnds, constraints=cons).x
print("GMV weights:", gmv.round(3), " vol:", round(np.sqrt(gmv@Sigma@gmv),3))
```
Drop `bnds` (allow shorting) and re-run with slightly perturbed `mu` to *watch* the weights explode — the instability lesson, live.
