## Block 1 — Walk downhill
To minimise a loss `L(θ)`, repeatedly step **against** the gradient (steepest-ascent direction):
```
θ ← θ − η · ∇L(θ)
```
`η` = **learning rate**. The gradient points uphill; we subtract it to descend. Repeat until the gradient ≈ 0 (a minimum).

*Why it matters:* when there's no formula for the optimum (neural nets, logistic regression, many calibrations), this is the universal fitting engine.

---

## Block 2 — The learning rate is everything
- Too small → painfully slow.
- Too large → overshoots, oscillates, or diverges.
- Practical fixes: **learning-rate schedules**, momentum, **Adam** (adaptive rates).

**Concept check:** what does the gradient being ≈0 tell you? (You're at a stationary point — minimum, maximum, or saddle.)

---

## Block 3 — Batch vs Stochastic (SGD)
- **Batch GD:** use all data per step — accurate gradient, expensive.
- **Stochastic GD (SGD):** one (or a *mini-batch* of) example per step — noisy but fast, and the noise can help escape shallow traps. SGD + mini-batches is the workhorse of deep learning.

---

## Block 4 — Mathematics (convex vs non-convex)
- **Convex** loss (e.g. OLS, ridge, logistic): one global minimum — GD finds it. Bowl-shaped, no traps.
- **Non-convex** (neural nets): many local minima/saddles — GD finds *a* good solution, not provably the best. Initialisation and randomness matter.

**Exercise:** minimise `L(θ)=(θ−3)²` by hand: `∇L=2(θ−3)`; from `θ=0, η=0.1` → `θ←0−0.1·(−6)=0.6 → …→ 3`. Watch it converge. **Challenge:** what happens with `η=1.1`? (Diverges — overshoots past the minimum each step.)

---

## Block 5 — Quant Thinking (where GD shows up on a desk)

- **Model calibration:** fitting a volatility surface or a stochastic-vol model = minimising pricing error via gradient methods.
- **Logistic/ML signals:** trained by GD on a loss.
- **Portfolio optimization:** often convex → solved by specialised convex solvers (the well-behaved cousin of GD).

**Hard puzzle:** your training loss keeps *increasing* each epoch. Most likely cause? (Learning rate too high → divergence. Lower `η`. Other suspects: bad data scaling, wrong sign.)

**Career connection:** "how would you fit a model with no closed-form solution?" → gradient descent (+ which flavour, learning-rate care, convex vs not). Knowing *why* it can fail (divergence, local minima) is the senior signal.

## Block 6 — Python (quant) · GD from scratch
```python
import numpy as np
rng = np.random.default_rng(0)
X = np.column_stack([np.ones(200), rng.normal(size=200)])
true = np.array([0.5, -1.2]); y = X@true + rng.normal(0,0.1,200)
theta = np.zeros(2); eta = 0.1
for _ in range(200):
    grad = (2/len(y)) * X.T @ (X@theta - y)     # gradient of MSE
    theta -= eta*grad
print(theta)   # ≈ [0.5, -1.2] — GD recovers the OLS solution
```
Bonus: try `eta=1.5` and watch it diverge.

**Linux:** `watch -n1 tail -5 train.log` to monitor a training loss live.
