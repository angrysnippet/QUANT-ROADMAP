## Block 2 — The microstructure picture (your mental model)
```
A trade happens when a taker (market order) meets a maker (limit order) in the book.
  → The spread the maker charges = processing + inventory + adverse-selection cost.
  → Large orders consume depth → temporary impact (recovers) + permanent impact (information).
  → To trade big: optimize the schedule (Almgren–Chriss: impact vs timing risk),
    choose order types / queue position / venues (SOR across lit & dark),
    and race latency where speed matters (HFT).
  → Measure everything after the fact with TCA (implementation shortfall).
  → All of it sits inside a regulated, fragmented, engineered market structure.
```
This is the complete loop from "I want to trade" to "here's what it actually cost and why."

## Block 3 — The mental models, distilled
- **There is no single price** — only a book; you buy at the ask, sell at the bid (181).
- **MMs earn the spread, fear inventory; skew quotes to stay flat** (Avellaneda–Stoikov, 182).

- **Impact = temporary (recoverable) + permanent (information)**; the √-law governs size (184).
- **Execution = mean–variance optimization across time** (Almgren–Chriss = Markowitz in time, 185).
- **Speed is an option on stale prices** — a relative, wasteful, real arms race (186).
- **Queue position and order type are tradeable edges** (187).
- **Measure with implementation shortfall, not VWAP-beating** (188).
- **The market is a fragmented, regulated, engineered system** (189).

## Block 4 — Mathematics · the relationships worth knowing cold
```
Effective spread:  2·|trade price − mid|   (and realized spread = − adverse selection)
Kyle's λ:          ΔPrice = λ·(net order flow);   depth = 1/λ
Square-root impact: ≈ Y·σ·√(Q/V)
Avellaneda–Stoikov: reservation r = s − q·γσ²(T−t)
Almgren–Chriss:    min  E[impact] + λ_risk·Var[timing];  risk-averse ⇒ front-load
Implementation shortfall: (avg exec − decision price)·shares + opportunity cost
```
If you can explain why the spread compensates adverse selection, write Kyle's λ, and map Almgren–Chriss onto Markowitz, you can reconstruct the module under pressure.

## Block 5 — Quant Thinking · interview rehearsal (say these out loud)
1. *Why does the bid–ask spread exist?* → processing + inventory + **adverse selection**; even a costless MM needs it to recoup losses to informed traders (181, 183).

3. *How do you execute a large order?* → Almgren–Chriss: trade off impact vs timing risk; risk aversion sets the front-loading — *mean–variance across time* (185).
4. *How does a market maker avoid getting run over?* → skew quotes against inventory, widen in volatility (Avellaneda–Stoikov), cancel stale quotes fast (182, 186).
5. *Is HFT good or bad?* → balanced: tighter spreads & liquidity vs phantom liquidity, arms-race waste, fairness — judge by mechanism and stress behavior (186).
6. *How do you measure execution quality?* → implementation shortfall vs decision price, decomposed; beware VWAP-gaming (188).
7. *Why does a stock trade on many venues?* → fragmentation via competition + Reg NMS; route smartly across lit/dark; prove best execution (189).

## Block 6 — Python (quant) · A unified mini execution simulator
```python
import numpy as np
rng = np.random.default_rng(0)

def execute(parent_qty, n_slices, urgency, adv, sigma, decision_px=100.0):
    """urgency in [0,1]: 0=TWAP(min impact), 1=front-loaded(min timing risk)."""
    # Almgren–Chriss-style schedule: front-load with urgency
    w = np.exp(-urgency*3*np.linspace(0,1,n_slices)); w/=w.sum()
    child = parent_qty*w
    px = decision_px; total_cost=0.0
    for q in child:
        impact = 0.1*sigma*np.sqrt(q/adv)              # √-law temporary impact
        px += sigma*np.sqrt(1/n_slices)*rng.standard_normal()  # price drift (timing risk)
        fill = px + impact                             # pay impact on top of drifted price
        total_cost += fill*q
    avg = total_cost/parent_qty
    IS_bps = (avg-decision_px)/decision_px*1e4
    return avg, IS_bps

for urg,label in [(0.0,"TWAP (slow)"),(0.5,"balanced"),(1.0,"front-loaded (fast)")]:
    avg,IS = execute(80_000, 20, urg, adv=1e6, sigma=0.02)
    print(f"{label:20s}: avg fill {avg:.4f}  implementation shortfall {IS:+.1f} bps")
```
The urgency knob slides between low-impact-but-drift-exposed (TWAP) and fast-but-high-impact (front-loaded) — the Almgren–Chriss trade-off and TCA measurement, unified in one simulator you fully understand.

**Linux:** `python execsim.py | tee exec.log`; raise `sigma` (more timing risk) and watch front-loading become relatively better — exactly the AC frontier shifting.
