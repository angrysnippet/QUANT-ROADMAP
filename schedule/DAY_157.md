## Block 1 — The one-step model
Over a small step, assume the stock can only go **up** to `S·u` or **down** to `S·d`. We replicate the option payoff with `Δ` shares + `B` cash and solve two equations (up-state, down-state) for two unknowns — *exactly* the replication logic of Day 152, now discrete. The result collapses to:
```
V = e^{−rΔt} · [ p·V_up + (1−p)·V_down ]      where   p = (e^{rΔt} − d)/(u − d)
```
`p` is the **risk-neutral probability** — note it depends only on `u, d, r`, **not** on the real probability of up/down or on `μ`. The drift vanished again (Day 153), now visibly. Standard calibration (Cox–Ross–Rubinstein): `u = e^{σ√Δt}`, `d = 1/u`.

*Why it matters:* this is risk-neutral valuation stripped to its skeleton — no calculus, just "discount the probability-weighted payoff under `p`."

---

## Block 2 — Building and pricing the multi-step tree
1. **Grow the tree forward:** from `S₀`, branch up/down for `N` steps → terminal prices `S₀·u^j·d^{N−j}`.
2. **Payoffs at the leaves:** `max(S_T − K, 0)` etc.
3. **Backward induction:** roll back node-by-node using `V = e^{−rΔt}[pV_up + (1−p)V_down]` until you reach today.

This **backward induction is dynamic programming** (Day 136's Bellman equation): the value at each node is the discounted expected value of its children — optimal substructure, solved right-to-left. Derivatives pricing *is* DP on a tree.

**Concept check:** with `N` steps, how many terminal nodes? (`N+1` — the tree *recombines* because `ud = du`, so it's a lattice, not an exponential blow-up. That recombination is what makes trees computationally cheap.)

---

## Block 3 — The tree's superpower: early exercise
For an **American** option you can exercise *any* time. The tree handles this trivially: at every node, take the **max** of continuing vs exercising now:
```
V_node = max( intrinsic_now ,  e^{−rΔt}[pV_up+(1−p)V_down] )
```
BS gives only the *European* price; the tree prices the early-exercise premium that BS can't. This is why trees remain a workhorse for American equity options, convertibles, and anything with embedded decision points — the `max` at each node is a **Bellman optimality** decision (Day 136), and the early-exercise boundary emerges naturally.

---

## Block 4 — Mathematics · convergence to Black–Scholes
As `N → ∞` (`Δt → 0`), the binomial tree **converges to the Black–Scholes price.** Intuition: a sum of `N` independent up/down log-steps → (Central Limit Theorem, Day 121) a Normal distribution of log-price → log-normal `S_T` → the BS integral. The CRR choice `u=e^{σ√Δt}` makes the per-step variance match `σ²Δt`, so the limiting distribution has exactly BS's variance. The convergence is *oscillatory* (even/odd `N` straddle the limit) — averaging adjacent `N`, or using a finer/`smoothed` tree, speeds it up. **The discrete tree and the continuous PDE are the same model viewed at two resolutions** — random walk → Brownian motion, all over again (Day 140).

---

## Block 5 — Quant Thinking · discretization is the universal numerical bridge
You rarely get closed forms in the wild (exotic payoffs, American features, stochastic vol). The tree teaches the master numerical pattern: **discretize the state, step through time, apply Bellman backward.** Its siblings — **finite-difference PDE solvers** (a grid instead of a tree) and **Monte Carlo** (Day 139, forward simulation for path-dependent payoffs) — are the three pillars of computational pricing. Choosing among them is real quant-dev judgment: trees/PDE for low-dimensional + early exercise; Monte Carlo for high-dimensional + path-dependent. **One conceptual toolkit (discretize + DP/simulate) prices essentially everything.**

**Hard puzzle:** an American *call* on a **non-dividend** stock — does the tree ever choose to exercise early? (No — at every node `continuation ≥ intrinsic` (the parity argument from Day 152: a live call beats exercising because you keep time value and earn interest on `K`). So the tree's American call price equals the European, confirming Day 152's theorem *numerically*. But add a dividend, or make it a **put**, and early exercise *does* fire — the tree shows you exactly when by where the `max` flips to "exercise.")

**Career connection:** "how would you price an American put?" Strong answer: **binomial tree** (or finite-difference PDE) — build the lattice, and at each node take `max(intrinsic, discounted risk-neutral continuation)`; that `max` captures the early-exercise premium BS misses, and it's literally backward-induction DP. Bonus points for noting it converges to BS for the European case and for choosing Monte Carlo instead when the payoff is path-dependent and high-dimensional.

## Block 6 — Python (quant) · A CRR tree pricing European *and* American puts
```python
import numpy as np

def binomial(S,K,r,sig,T,N,kind="put",american=False):
    dt = T/N
    u  = np.exp(sig*np.sqrt(dt)); d = 1/u
    p  = (np.exp(r*dt)-d)/(u-d); disc = np.exp(-r*dt)
    # terminal prices and payoffs
    j  = np.arange(N+1)
    S_T = S*u**j*d**(N-j)
    V  = np.maximum(K-S_T,0) if kind=="put" else np.maximum(S_T-K,0)
    # backward induction
    for i in range(N-1,-1,-1):
        j = np.arange(i+1)
        S_i = S*u**j*d**(i-j)
        V = disc*(p*V[1:i+2] + (1-p)*V[0:i+1])
        if american:
            exer = (K-S_i) if kind=="put" else (S_i-K)
            V = np.maximum(V, np.maximum(exer,0))
    return V[0]

S,K,r,sig,T = 100,100,0.05,0.2,1.0
print("European put  (tree, N=500):", round(binomial(S,K,r,sig,T,500,"put",False),4))
print("American put  (tree, N=500):", round(binomial(S,K,r,sig,T,500,"put",True ),4))
# convergence to BS for the European call
for N in (10,50,200,1000):
    print(f"N={N:5d}  European call = {binomial(S,K,r,sig,T,N,'call',False):.4f}")
```
The American put is worth **more** than the European (the early-exercise premium), and the European call marches toward the BS price ≈10.4506 as `N` grows — DP-on-a-tree pricing, fully visible.

**Linux:** `python tree.py | tee tree.log`; bump `N` and watch the oscillatory convergence; flip `american=True` on the call (no dividend) to confirm it equals the European.

Discretization isn't an approximation to apologize for; it's how you compute anything the continuous math can't solve in closed form. **When stuck analytically, discretize and recurse.**
