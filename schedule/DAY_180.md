## Block 1 — The one idea: stationary structure + breadth
Every day of this module served one thesis: **prices are unpredictable, but certain *combinations and cross-sectional rankings* are not — and a portfolio of many such weak, decorrelated bets is a reliable edge.**
- **Find the structure:** cointegration / residual signals (172, 176).
- **Model it:** OU process, half-life, Kalman (173, 175).
- **Aggregate it:** breadth via the Fundamental Law `IR = IC·√Breadth` (176).
- **Combine it:** orthogonalize many weak signals (177).
- **Survive it:** crowding, leverage, capacity, costs (178, 179).

If you keep one thing from Phase 10: **the edge is never one brilliant bet — it's the Law of Large Numbers, industrialized, and defended against the risks your backtest can't see.**

## Block 2 — The stat-arb pipeline (your mental flowchart)
```
1. Universe & candidate relationships (economic rationale first).
2. Test for stationarity/cointegration (Engle–Granger / Johansen); reject spurious.
3. Model reversion: OU → half-life → entry/exit; Kalman for a dynamic hedge ratio.
4. Cross-section: build signals, NEUTRALIZE vs factors, rank, form dollar-neutral book.
5. Combine signals by marginal/orthogonal contribution (+ shrinkage).
6. Backtest HONESTLY: lookahead-safe, survivorship-clean, net of costs, walk-forward, trial-deflated.
7. Risk: size for crowding/stress, cap leverage & turnover, respect capacity.
8. Execute: minimize impact (√-law), net trades, schedule.
```
This routes essentially any mean-reversion / cross-sectional strategy you'll build.

## Block 3 — The mental models, distilled
- **Correlation ≠ cointegration** (171–172): test the *spread* for stationarity, not returns for correlation.
- **OU half-life** = `ln2/θ` sets the holding period and turnover (173).
- **Kalman** = optimal adaptive hedge ratio; predict–update = real-time Bayes (175).
- **`IR = IC·√Breadth`** — breadth (if independent) is the engine; neutralize to keep it independent (176).
- **A signal's value is marginal/orthogonal, not standalone** (177).
- **Crowding + leverage + forced selling = the Quant Quake**; the worst risk is unmodeled (178).
- **Square-root impact + capacity**: alpha is finite in time *and* size; judge it net, at scale (179).

## Block 4 — Mathematics · the equations worth knowing cold
```
Cointegration:   ε_t = A_t − β B_t  is I(0)  (ADF rejects unit root)
OU spread:       dx = θ(μ − x)dt + σ dW;   half-life = ln2/θ;  Var = σ²/2θ
Kalman update:   β ← β_pred + K(obs − pred)
Fundamental Law: IR ≈ IC · √Breadth
Net return:      gross alpha − turnover × cost
Market impact:   ≈ k·σ·√(Q/V);   capacity: net profit ∝ alpha·$ − c·$^{1.5}/√V
```
If you can state cointegration, derive the half-life, write the Fundamental Law, and explain the net-return/capacity trade-off, you can reconstruct the module under interview pressure.

## Block 5 — Quant Thinking · interview rehearsal (say these out loud)
1. *How does pairs trading work and why market-neutral?* → trade the cointegrated spread on z-score; long-A/short-B cancels market beta → pure idiosyncratic alpha (171).
2. *Correlation or cointegration — which do you test, and why?* → cointegration: the *spread* must be stationary; high return-correlation can still random-walk apart (172).
3. *How do you set the holding period?* → OU half-life (`ln2/θ`); trade off edge-per-trade vs turnover/costs (173).
4. *How does a big fund profit from coin-flip predictions?* → cross-sectional, factor-neutral breadth: `IR = IC·√Breadth` over thousands of *independent* names (176).
5. *You have 10 signals — combine them?* → portfolio-of-signals: orthogonalize, shrink, weight by marginal contribution; equal-weight is a tough OOS baseline (177).
6. *Market-neutral, great Sharpe — what's the real risk?* → crowding + forced deleveraging at leverage (the 2007 Quake); size for the unmodeled stress (178).
7. *Before scaling AUM, first check?* → costs & capacity: net of square-root impact, find where alpha meets impact (179).

**Hard puzzle (capstone):** you discover two cointegrated stocks with a 5-day half-life, a backtested net Sharpe of 2.5, on liquid large-caps. Walk through *everything* you'd verify before allocating \$200M. (1) **Cointegration is real & persistent** — holds out-of-sample, not a multiple-testing fluke, with an *economic* tether (same sector/business), and re-tested on a rolling basis for breakdown. 2) **OU model sane** — 5-day half-life implies ~weekly round-trips; confirm θ stable, set z-thresholds and a *time-stop ≫ half-life* + divergence stop for breakdown (174). 3) **Backtest honesty** — lookahead-safe (`.shift(1)`), survivorship-clean (what if a leg gets acquired/delisted?), **net of realistic costs**, walk-forward, deflated for trials (174). 4) **Breadth/concentration** — *one* pair is fragile; is this part of a *diversified book* of hundreds of independent pairs, or a single concentrated bet? `IR=IC·√Breadth` says one pair ≠ a business (171, 176). 5) **Crowding** — is this spread obvious enough that peers hold it; what happens in a deleveraging unwind (178)? 6) **Capacity** — at \$200M, what's `Q/V` per leg; does square-root impact survive my turnover (179)? 7) **Leverage/sizing** — size for the stress scenario, not the backtest Sharpe (178). The point: a 2.5 Sharpe on one pair is the *start* of due diligence, not the end — this single question touches all ten days.)

## Block 6 — Python (quant) · A unified mini stat-arb toolkit
```python
import numpy as np, pandas as pd
from statsmodels.tsa.stattools import coint, adfuller

class StatArb:
    def __init__(self, A, B): self.A, self.B = pd.Series(A), pd.Series(B)
    def cointegrated(self): return coint(self.A, self.B)[1]          # p-value
    def hedge_ratio(self):  return np.polyfit(self.B, self.A, 1)[0]
    def spread(self):       return self.A - self.hedge_ratio()*self.B
    def half_life(self):
        s = self.spread(); ds, sl = s.diff().dropna(), s.shift(1).dropna()
        b = np.polyfit(sl.values[1:], ds.values, 1)[0]
        return -np.log(2)/b
    def zscore(self, lb=30):
        s = self.spread(); return (s - s.rolling(lb).mean().shift(1))/s.rolling(lb).std().shift(1)

rng = np.random.default_rng(0); T=1000
common = np.cumsum(rng.standard_normal(T)*0.01); sp=np.zeros(T)
for t in range(1,T): sp[t]=0.93*sp[t-1]+rng.standard_normal()*0.4
A = 100*np.exp(common+0.01*sp); B = 100*np.exp(common)

sa = StatArb(A, B)
print(f"cointegration p-value : {sa.cointegrated():.4f}")
print(f"hedge ratio β         : {sa.hedge_ratio():.3f}")
print(f"half-life (days)      : {sa.half_life():.1f}")
print(f"latest z-score        : {sa.zscore().iloc[-1]:.2f}")
```
Cointegration test, hedge ratio, half-life, and a lookahead-safe z-score — the whole module in one class you fully understand.

**Linux:** `python statarb.py | tee statarb.log`; extend it with the OU optimal thresholds and a cost-aware backtest — you now have every tool to do so.
