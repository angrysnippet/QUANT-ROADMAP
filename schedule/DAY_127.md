## Block 1 — Memorising ≠ learning
A model **overfits** when it captures noise specific to the training data rather than the underlying signal. Symptom: **great in-sample, poor out-of-sample.** A flexible-enough model can fit *any* historical price path perfectly — and predict nothing.

*Why it matters:* markets have a **tiny signal-to-noise ratio.** Flexibility that helps in image recognition is poison in finance — there's far more noise to memorise.

---

## Block 2 — Train / Validation / Test
- **Train:** fit parameters.
- **Validation:** tune hyperparameters / choose models.
- **Test:** touched **once**, at the end, to estimate true performance.

Reuse the test set and it silently becomes training data — your estimate inflates. **Cross-validation** (k-fold) reuses data for validation, but in time series you must use **forward/blocked CV** (never train on the future — Day 124's lookahead).

---

## Block 3 — Detecting & fighting overfitting
- **In-sample ≫ out-of-sample** performance → overfit.
- **Too many parameters** relative to (effective) data points.
- **Sensitivity:** tiny data changes swing the result wildly.

Defences: **regularization** (L1/L2 penalties — keep weights small/sparse), **simpler models**, **more data**, **fewer trials** (Day 125), and **walk-forward** testing.

**Concept check:** why does fitting 20 parameters on 30 noisy data points almost guarantee overfit? (Degrees of freedom ≈ data → it interpolates noise.)

---

## Block 4 — Mathematics (regularization)
Ridge (L2) regression minimises `‖y − Xβ‖² + λ‖β‖²`: the penalty `λ‖β‖²` shrinks coefficients toward 0, trading a little bias for a lot less variance. Lasso (L1, `λ‖β‖₁`) drives some coefficients to exactly 0 (feature selection). **`λ` controls the bias-variance dial** (Day 128).

**Exercise:** as `λ → ∞`, what happens to the fitted model? (All coefficients → 0 → predicts the mean → maximally biased, minimally variable.)

---

## Block 5 — Quant Thinking (Backtest overfitting is the norm, not the exception)

- Every tuned threshold, lookback window, and asset filter is a **parameter** — collectively a giant overfit risk.
- **The more beautiful the backtest, the more suspicious.** A Sharpe of 4 from retail-accessible data is almost certainly overfit or leaked.
- **Deflated Sharpe / PBO** (probability of backtest overfitting, López de Prado): formal tools to discount results by trials run.
- **The cure is culture:** few hypotheses, economic rationale, locked tests, walk-forward, and humility.

**Hard puzzle:** your strategy uses a 23-day moving average and works great; 22-day and 24-day fail. Real signal or overfit? (Overfit — a real effect is **robust to nearby parameters**; cliff-edge sensitivity to one magic number screams curve-fitting.)

**Career connection:** "how do you know you're not overfitting?" is asked in *every* quant-research interview. Strong answer: out-of-sample/walk-forward, parameter robustness, few trials, regularization, deflated Sharpe.

## Block 6 — Python (quant) · Watch it overfit
```python
import numpy as np
rng = np.random.default_rng(0)
x = np.linspace(0,1,30); y = np.sin(2*np.pi*x) + rng.normal(0,0.3,30)
for deg in [1, 3, 15]:
    c = np.polyfit(x, y, deg)
    train_err = np.mean((np.polyval(c,x)-y)**2)
    xt = np.linspace(0,1,200); yt = np.sin(2*np.pi*xt)
    test_err = np.mean((np.polyval(c,xt)-yt)**2)
    print(deg, round(train_err,3), round(test_err,3))   # deg15: train↓ test↑
```
Degree-15 nails the training points and explodes on new data — overfitting in one screen.

**Linux:** version-control every experiment (`git`) so you can see how many variants you tried — your honest trial count.

## Quant Thinking Track — Generalization Is the Only Goal
In-sample performance is *free and meaningless.* The entire game is **out-of-sample generalization.** Anything that doesn't help on unseen data is, at best, decoration.
