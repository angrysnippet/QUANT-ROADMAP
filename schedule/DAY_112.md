## Why today matters
*"How do I get from state A to state B optimally?"* appears in routing, scheduling, allocation, portfolios, research planning, learning. The difference isn't *optimization* — it's *what structure the optimization has.*

---

## Block 1 — The grand unification
Every optimization problem is **state → decision → new state.** Learning (knowledge → study → new knowledge), portfolio (holdings → trade → new holdings), graph (node → move → new node). Optimization = *choosing good transitions between states.*

*Why it matters:* one mental model covers three "separate" topics — you select by structure, not by memory.

---

## Block 2 — DSA · When Greedy works
Greedy asks *what's the best immediate move?* (Activity Selection → earliest finish). Works when *local best = global best.* **Key question:** can I *prove* best-now = best-overall? Signals: intervals, scheduling, sorting, provably-safe local choice.

---

## Block 3 — DSA · When DP works
DP asks *what future consequences does this choice create?* (Knapsack: take/skip changes future capacity → greedy can fail). **Key question:** does the current choice change future opportunities? Signals: take/skip, max/min, count ways, future matters.

---

## Block 4 — DSA · When Shortest Path works
`A → B → C` with costs → *cheapest route?* = path optimization. **Key question:** can it be represented as nodes + weighted transitions? Signals: route, cost, distance, travel, network.

---

## Block 5 — Mathematics (One framework)
With *state → transition → cost*: Shortest Path minimises total cost; DP optimises state values; Greedy picks the best local transition. All solve **optimization over states.**

| Technique | Core idea |
|-----------|-----------|
| Greedy | Best local transition |
| DP | Best future-aware transition |
| Shortest Path | Best path through a graph |
| Graph DP | DP on a DAG |
| Dijkstra | Cost-based state exploration |

**Exercise:** classify "cheapest flight", "max reward under budget", "max meetings."

## Block 6 — Quant Thinking (Optimization everywhere)

Most quant work is optimization with trade-offs: portfolio (return↑/risk↓), execution (speed↑/cost↓), research (impact↑/time↓). Algorithms navigate trade-offs.

**Hard puzzle:** 3 years + limited time/energy, goal = quant-ready — Greedy, DP, or Shortest Path? Think deeply (no single perfect answer — it's a constrained sequential-decision problem).

**Career connection:** strong engineers don't memorise 200 algorithms — they recognise **optimization structure**, then pick Greedy / DP / Graph / Search.
