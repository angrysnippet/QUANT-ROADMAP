## Why today matters
Beginner: *problem → find algorithm → solve.* Advanced: *problem → break into components → solve each → combine.* Most hard problems are **Algorithm A + Algorithm B**, not A alone.

---

## Block 1 — C++ (Pattern combination thinking)
`Graph + weights → shortest path` = Dijkstra (easy). But `Graph + shortest path + a reward constraint` may need **more than one** algorithm working together.

*Why it matters:* the leap from intermediate to advanced is composing known tools, not learning new ones.

**Exercise:** `Graph + optimization` — which algorithms might interact?

---

## Block 2 — DSA · Common Interview Combinations
- **Binary Search + Greedy** — "minimum/maximum possible", "optimize answer" → binary-search the answer, greedily check feasibility.
- **Graph + DP** — dependencies/rewards/longest chain → topological sort + Graph DP.
- **DP + Optimization** — budget/resources/max value → Knapsack family.
- **Graph + Shortest Path** — cities/roads/routing → BFS or Dijkstra.

---

## Block 3 — DSA · Decomposition Framework
Three questions for any hard problem:
1. **Objects** — array? graph? grid? string?
2. **Objective** — count? max? min? search?
3. **Constraints** — budget? dependencies? capacity? obstacles?

*Object + Objective + Constraint* often reveals the solution.

**Exercise:** road network + shortest travel time + weighted roads → object **Graph**, objective **Minimum**, algorithm **Dijkstra.**

---

## Block 4 — Mathematics (Building models)
Strong solvers think *model* first, *code* second. Portfolio optimization models as *items + value + risk + budget* — suspiciously like **Knapsack.**

**Challenge:** why is modelling often harder than implementation?

---

## Block 5 — Quant Thinking (Reality → abstraction)

100 stocks + limited capital + target return → *items / weight / value / capacity* → **Knapsack.** Research dependencies → **DAG.** Route optimization → **shortest path.**

**Hard puzzle:** your roadmap (C++, DSA, Probability, Statistics, ML, Projects) — some topics depend on others, some give higher value. Model it as Graph, DP, or **Graph DP**? Explain.

**Career connection:** quant researchers spend enormous time **modelling** — wrong model → wrong algorithm → wrong answer. The strongest people are the best *modelers*, not the fastest coders.
