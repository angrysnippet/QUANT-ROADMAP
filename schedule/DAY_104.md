## Block 1 — C++ · Pattern Revision
Create `binary_search_pattern.cpp` — write the template from memory:
```cpp
bool check(int mid) { /* feasibility */ }
while (low <= high) { int mid = (low+high)/2; if (check(mid)) ... }
```

*Why it matters:* once the *template* is automatic, every new problem reduces to "define the answer space and the check."

---

## Block 2 — DSA · Universal Framework
1. Define the **answer space.**
2. Define **`check(X)`.**
3. Find the **monotonic property.**
4. Binary search.

- **Aggressive Cows:** answer = distance; check = can cows be placed?; small works / large fails.
- **Allocate Books:** answer = max pages; check = can books be assigned?; small fails / large works.

---

## Block 3 — DSA · Recognition Signals
Keywords *minimum possible / maximum possible / optimize answer* → stop → ask *can I check whether X works?* → *does feasibility change only once?* → **binary search on answer.**

**Drill:** (A) min truck capacity to deliver packages → BS answer; (B) maximise the minimum distance → Aggressive Cows family; (C) min time to finish tasks → BS answer.

---

## Block 4 — Mathematics (Monotonic functions)
`F F F T T` has a boundary → binary search works. It's a step function `f(x)=0 for x<k, 1 for x≥k`.

**Exercise:** would binary search work for `T F T F`? Why not? **Challenge:** why is monotonicity the *real* reason binary search works (not sortedness)?

## Block 5 — Quant Thinking (Optimization through constraints)

*Minimum servers required?* — don't solve directly; check "can 10 / 20 / 30 handle the load?", then search.

**Problems:** cloud capacity planning; warehouse logistics; trading infrastructure.

**Hard puzzle:** minimum daily study hours to finish the roadmap within 3 years — define `check(hours)`.

**Career connection:** resource planning, capacity allocation, scheduling, risk limits, infra design rarely compute the best answer directly — they repeatedly ask *can this work?* Converting *optimization → feasibility* is a powerful engineering skill.

## Quant Thinking Track — Searching for the Boundary
Many optimization problems are really *find the boundary* problems, not *find the answer directly*: Aggressive Cows = largest distance that still works; Allocate Books = smallest limit that still works. The target is **the boundary between possible and impossible.**
