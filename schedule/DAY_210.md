## Block 1 — The big picture: where the structure lives
The module's organizing insight: **financial time series have structure in specific, identifiable places — find it where it lives, and ignore where it doesn't.**
- **conditional mean** of returns → ~no structure (ARIMA finds nothing, Day 201; efficiency, Day 150),
- **conditional variance** → strong, forecastable structure (GARCH, Day 202) — *the* forecastable thing,
- **relationships between series** → cointegration/lead–lag (VAR/VECM, Days 205–206) — relative-value alpha,
- **regimes** → discrete switching structure (HMM, Day 204) — explains decay and breakdown,
- **cycles/seasonality** → frequency-domain structure (spectral/wavelets, Day 207) — mostly in volume/vol, not returns,
- **hidden states from many signals** → factor/state-space structure (DFM/Kalman, Days 203, 209) — real-time estimation.

Keep one thing from Phase 13: **don't model time series generically — identify *where* the forecastable structure is (variance, relationships, regimes, cycles — rarely the return mean) and bring the matching tool.**

## Block 2 — The deep unities (this module's gift)
Three unifications make the whole toolkit cohere:
1. **State-space is the master framework** (Day 203): ARIMA, exponential smoothing, GARCH-as-stochastic-vol, dynamic regression, factor models, and DFMs are *all* special cases — "a hidden state evolving, observed through noise," solved by *one* engine (Kalman). Learn the predict–update loop once, apply everywhere.
2. **"Few drivers, many observables" recurs everywhere** (low-rank structure): PCA on curves (166), factor models for returns (148), cointegration common-trends (206), dynamic factor models (209) — *all* the same idea that high-dimensional financial data is *effectively low-dimensional*. Exploit the low rank.
3. **Decompose into (unpredictable common part) + (tradeable residual)**: market factor + idiosyncratic (176), common trends + cointegrating residual (206), trend + cycle + noise (spectral, 207) — hedge the common part, trade/model the residual.

These three unities turn a "zoo of techniques" into one coherent way of seeing.

## Block 4 — Mathematics · the relationships worth knowing cold
```
ARIMA(p,d,q):  AR + I (differencing, Day 142) + MA
GARCH(1,1):    σ²_t = ω + αε²_{t−1} + βσ²_{t−1}   (persistence α+β)
State-space:   state_t = F·state_{t−1} + noise;  obs_t = H·state_t + noise  (Kalman)
HMM:           hidden regime ~ Markov chain; per-regime emissions; EM/Baum–Welch
VECM:          Δx_t = α·(β'x_{t−1}) + lags + ε   (error correction)
Forecast test: beat random-walk/mean benchmark; Diebold–Mariano significance
```
If you can explain why ARIMA finds nothing in returns, write GARCH, cast a model in state-space, and state the cointegration = factor-decomposition unity, you can reconstruct the module under pressure.

## Block 5 — Quant Thinking · interview rehearsal (say these out loud)
1. *Forecast stock returns with ARIMA?* → it models the mean, which is ~unpredictable; ARIMA(0,0,0) is a correct null → model variance (GARCH) and relative value instead (201).
2. *What's actually forecastable in markets?* → not return direction, but variance — volatility clusters/persists; GARCH/EGARCH (202).
3. *Estimate a drifting beta in real time?* → state-space + Kalman filtering (lookahead-safe), missing data handled by skipping the update (203).
4. *Why do strategies suddenly stop working?* → regime change; model with HMM/Markov-switching, adapt on filtered regime probability (204).
5. *Does X cause Y?* → Granger causality tests *predictability*, not causation; run on stationary data; beware omitted variables (205).
6. *Find relative value among many assets?* → Johansen/VECM cointegration = factor-neutralization in level-space (206).
7. *How do you know a forecast is good?* → beats the naïve benchmark, walk-forward, significantly (Diebold–Mariano); be willing to conclude no edge (208).
8. *Estimate the economy's current state in real time?* → nowcasting via state-space dynamic factor model; vintage data; fuse mixed-frequency signals (209).

**Hard puzzle (capstone):** you're asked to build a complete system that (a) forecasts each asset's *risk* for position sizing, (b) detects market regimes for risk-on/off, (c) finds relative-value trades among related assets, and (d) gives a real-time read on macro conditions — all from messy, mixed-frequency data. Architect it using the whole module. (A unified **state-space-centric** architecture, since state-space + Kalman is the master framework (Day 203) that ties it all together: **(a) Risk forecasting** → fit **GARCH/EGARCH** (Day 202) per asset on return residuals (after a thin ARIMA mean model, Day 201) to forecast the **variance term structure**; feed forecast vol into **vol-targeting position sizing** (Day 143) — risk is *the* forecastable quantity, so size on it. Use the **density forecast** (interval), not a point (Day 208). **(b) Regime detection** → a **Hidden Markov / Markov-switching model** (Day 204) on returns/vol (and the macro factor from (d)) to estimate the **filtered** (lookahead-safe) probability of calm/crisis regimes; scale gross risk-on/off exposure by it, and switch risk models (correlations are regime-dependent, Day 146). **(c) Relative value** → **Johansen/VECM** (Day 206) on related-asset baskets/curves to find **cointegrating relationships** (the tradeable mean-reverting residuals after hedging out the common trends — factor-neutralization in level-space), with **OU half-lives** (Day 173) for sizing/holding and rolling re-tests for breakdown (Day 174). **(d) Macro nowcast** → a **state-space dynamic factor model** (Day 209) fusing ~100s of **mixed-frequency** indicators (using **vintage/point-in-time** data to avoid revision leakage, Day 141) into a few real-time macro factors via the **Kalman filter**, feeding both the regime model (b) and risk-on/off. **The unifying architecture:** *everything is a hidden state observed through noisy signals, estimated in real time by Kalman/state-space* (Day 203) — the GARCH variance, the HMM regime, the cointegration equilibrium, and the macro factors are all hidden states; the **predict–update loop** is the common engine. **Cross-cutting discipline:** (1) **honest evaluation** (Day 208) — walk-forward, beat naïve benchmarks, deflate for the search; (2) **lookahead safety** everywhere (filtering not smoothing, vintage data, `.shift(1)` — Day 141); (3) **parsimony** (few factors, 2–3 regimes — overfitting, Day 191); (4) **regime-awareness of correlations** (Day 146) and **breakdown risk** (Day 174). **The synthesis:** this is the entire module assembled into one coherent system, unified by the state-space paradigm — *fuse noisy signals into real-time estimates of the hidden states that matter (risk, regime, equilibrium, macro), and act on them with honest validation and lookahead discipline.* That integration — not any single model — is the capstone skill of advanced time series.)

## Block 6 — Python (quant) · The integrated time-series stack
```python
import numpy as np, pandas as pd
rng = np.random.default_rng(0)
T=1500; ret = rng.standard_normal(T)*0.01

# (a) RISK: EWMA/GARCH-style variance forecast for position sizing
vol = pd.Series(ret).pow(2).ewm(span=40).mean().pow(0.5).shift(1)   # lookahead-safe
target = 0.01
size = (target/vol).clip(upper=3)                                    # vol-targeted size

# (b) REGIME: simple 2-state vol regime (proxy for HMM filtered prob)
hi_vol_regime = (vol > vol.median()).astype(int)                    # crisis proxy

# (c) RELATIVE VALUE: a cointegrated pair's z-score signal (Day 206 in miniature)
common = np.cumsum(rng.standard_normal(T)*0.01)
A = np.exp(common + 0.01*np.cumsum(rng.standard_normal(T)*0.3))
B = np.exp(common)
spread = np.log(A) - np.log(B)
z = (spread - pd.Series(spread).rolling(40).mean().shift(1)) / pd.Series(spread).rolling(40).std().shift(1)

print(f"avg vol-targeted size      : {size.mean():.2f}")
print(f"fraction in high-vol regime: {hi_vol_regime.mean():.2f}")
print(f"relative-value |z|>2 days  : {(np.abs(z)>2).sum()}")
print("→ one stack: forecast risk (a), detect regime (b), trade relative value (c) — all lookahead-safe")
```
The integrated stack — forecast risk for sizing, detect the regime, and trade relative value — all lookahead-safe, all from the same time-series toolkit. A miniature of the capstone architecture.

**Linux:** `python ts_stack.py | tee stack.log`; this scaffold (vol-target + regime + relative-value, with `.shift(1)` everywhere) is the skeleton of a real systematic time-series system.
