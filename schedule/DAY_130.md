## Block 1 — The model
```
y = β₀ + β₁x₁ + … + βₖxₖ + ε
```
Fit by **Ordinary Least Squares**: choose `β` to minimise `Σ(yᵢ − ŷᵢ)²`. Closed form (normal equations):
```
β̂ = (XᵀX)⁻¹ Xᵀy
```
*Why it matters:* it's interpretable, fast, regularizable, and — in low-SNR markets — usually the *right* complexity (Day 128).

---

## Block 2 — Interpreting the output
- **Coefficient `βⱼ`:** expected change in `y` per unit `xⱼ`, holding others fixed.
- **t-stat of `βⱼ`** `= β̂ⱼ / SE(β̂ⱼ)`: is this predictor real? (|t|>2 ≈ significant — Day 125).
- **R²:** fraction of variance explained. In finance, **tiny R² (1–5%) can still be very profitable** — don't chase high R².

**Concept check:** a signal has R² = 0.01 but a stable, significant positive coefficient. Tradeable? (Possibly very — small predictive power over many independent trades compounds into Sharpe.)

---

## Block 3 — The OLS assumptions (and how markets violate them)
1. **Linearity** — relationship is linear (often only approximately).
2. **Independence** of errors — *violated* by autocorrelation → use Newey–West/HAC standard errors.
3. **Homoscedasticity** — constant error variance — *violated* by volatility clustering (heteroscedasticity).
4. **No multicollinearity** — correlated features inflate coefficient variance (unstable βs).
5. **Normal errors** (for exact inference) — returns are fat-tailed.

**Exercise:** two features correlate 0.98. What breaks, and what's a fix? (Multicollinearity → unstable, huge-variance coefficients → drop one, combine them, or use ridge.)

---

## Block 4 — Mathematics (regression = projection)
OLS geometrically **projects** `y` onto the span of the feature columns; residuals are orthogonal to features. Ridge adds `λ‖β‖²` → shrinks β (Day 127), trading bias for variance — essential when features are collinear or numerous.

## Block 5 — Quant Thinking (regression *is* the desk)

- **Factor models:** `r_stock = α + β_mkt·r_mkt + β_size·SMB + β_value·HML + … + ε`. The βs are **factor exposures**; the residual `ε` is the part not explained by known risks.
- **Alpha = the intercept `α`** after controlling for factors — *skill beyond exposure* (Day 149).
- **Hedging:** regress your position's returns on a hedge instrument; the slope `β` is the **hedge ratio** (how much to short to neutralise risk).
- **Beta-neutralisation:** subtract `β·factor` to isolate residual signal (Day 124's "control for confounders").

**Hard puzzle:** you regress a stock on the market and get `β = 1.4, α = 0.02% /day, R² = 0.45`. What does each number tell a PM? (β: 40% more market-sensitive than average → hedge 1.4 units; α: small daily outperformance vs market — test its t-stat; R²: ~45% of moves are market-driven, 55% idiosyncratic.)

**Career connection:** regression appears in *every* quant role — "how would you estimate a hedge ratio / isolate alpha / combine signals?" The answer is almost always *a (regularized) regression* — and knowing its assumptions/violations is what separates juniors from seniors.

## Block 6 — Python (quant) · OLS + factor regression
```python
import numpy as np
rng = np.random.default_rng(0)
mkt = rng.normal(0, 0.01, 1000)
stock = 0.0002 + 1.4*mkt + rng.normal(0, 0.012, 1000)   # alpha + beta*mkt + idio
X = np.column_stack([np.ones_like(mkt), mkt])
beta = np.linalg.lstsq(X, stock, rcond=None)[0]
print("alpha/day:", round(beta[0],5), " beta:", round(beta[1],3))
resid = stock - X@beta
r2 = 1 - resid.var()/stock.var(); print("R^2:", round(r2,3))
```
Bonus: add a second factor; use `statsmodels` to get t-stats and HAC (Newey–West) standard errors.

**Linux:** `wc -l prices.csv`, `head` — always sanity-check data dimensions before regressing.

## Quant Thinking Track — Small, Stable, Significant
In finance, prize **stable, significant** coefficients with **small** R² over flashy high-R² fits (usually overfit/leaked). Persistence beats magnitude.
