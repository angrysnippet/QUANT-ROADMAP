## Block 1 — The two organizing ideas
The whole module rests on two themes that recur in every day:
1. **Robustness over optimality** — naïve optimization fits noise and fails out-of-sample; real construction *builds for input uncertainty* via structure (factor models, 211), priors (Black–Litterman, 212), shrinkage/constraints (robust opt, 218), and estimable-only inputs (risk parity, 214). *Accept in-sample suboptimality for out-of-sample survival.*
2. **Survival over return-maximization** — the binding constraint is *staying in the game*: size for survival (fractional Kelly, 213), measure tail risk honestly (CVaR/EVT, 215–216), control drawdown (217), and respect that ruin is absorbing. *Optimize the path you can survive, not the highest expected return.*

Keep one thing from Phase 14: **build robust portfolios that survive — respect estimation error and the tail, size for survival, and always know where your returns come from.**

## Block 2 — The portfolio pipeline (your mental flowchart)
```
1. Inputs: estimate Σ via a FACTOR MODEL (211, structured/stable); be humble about μ.
2. Expected returns: anchor on an equilibrium PRIOR, tilt with confidence-weighted views (Black–Litterman, 212).
3. Allocation: optimize ROBUSTLY (shrinkage + constraints, 218) — or go risk-based (risk parity, 214) to avoid μ entirely.
4. Sizing/leverage: FRACTIONAL KELLY (213) — size for the edge you have, with a survival margin.
5. Risk measurement: CVaR/Expected Shortfall (215) + EVT for the tail (216), not just σ/VaR. Stress-test.
6. Drawdown control: vol-targeting, diversification, tail hedges (217) — optimize the holdable path.
7. Attribution: decompose realized returns — factor vs alpha, skill vs luck (219). Learn & improve.
```
This is how institutions actually build and manage portfolios — robust, survival-oriented, and self-auditing.

## Block 4 — Mathematics · the relationships worth knowing cold
```
Factor risk:    Σ = B·F·Bᵀ + D   (systematic + specific)
Black–Litterman: E[R] = precision-weighted blend of equilibrium prior Π and views Q
Kelly:          f* = μ/σ²;  growth g(f) = fμ − ½f²σ²  (zero at 2f*)
Risk parity:    equalize RC_i = w_i·(Σw)_i/σ_p
CVaR:           E[Loss | Loss > VaR]  (coherent; ≥ VaR)
EVT:            tail ~ power law x^{−α}; tail index ξ > 0 for finance
Drawdown:       recovery from −d needs +d/(1−d)
```

## Block 6 — Python (quant) · The integrated robust-portfolio stack
```python
import numpy as np
from sklearn.covariance import LedoitWolf
rng = np.random.default_rng(0)

N, T = 20, 150
sample = rng.multivariate_normal(rng.uniform(0.03,0.10,N)/252,
                                 (np.diag(rng.uniform(0.02,0.06,N))+0.008)/252, T)

# 1. robust Σ (shrinkage); 2. equilibrium prior; 3. risk-parity sizing; 4. fractional Kelly; 5. CVaR
Sigma = LedoitWolf().fit(sample).covariance_*252
w_mkt = np.ones(N)/N
Pi = 2.5 * Sigma @ w_mkt                                  # equilibrium prior returns (BL anchor)

# risk-parity weights (uses only Σ) as a robust base
from scipy.optimize import minimize
def rp_obj(w):
    sig=np.sqrt(w@Sigma@w); rc=w*(Sigma@w)/sig; return np.sum((rc-rc.mean())**2)
w_rp = minimize(rp_obj, w_mkt, bounds=[(0,1)]*N,
                constraints=({'type':'eq','fun':lambda w:w.sum()-1},)).x

# fractional Kelly leverage to a vol target, then risk metrics
port_ret = sample @ w_rp
mu, sig = port_ret.mean()*252, port_ret.std()*np.sqrt(252)
kelly = mu/sig**2; lev = 0.5*kelly                        # HALF-Kelly (survival margin)
scaled = lev*port_ret
cvar = -scaled[scaled <= np.quantile(scaled,0.05)].mean()  # 95% CVaR (tail)
equity = np.cumprod(1+scaled); maxdd = (equity/np.maximum.accumulate(equity)-1).min()
print(f"risk-parity + half-Kelly: leverage {lev:.2f}, ann.return {mu*lev:.1%}, vol {sig*lev:.1%}")
print(f"95% CVaR (daily tail): {cvar:.2%}   max drawdown: {maxdd:.1%}")
print("→ robust Σ + risk-parity base + half-Kelly sizing + tail-aware risk = the integrated stack")
```
The integrated stack — shrunk Σ, an equilibrium-anchored risk-parity base, half-Kelly sizing for survival, and CVaR/drawdown tail monitoring — is how the module's pieces fit into one robust, survival-oriented portfolio process.
