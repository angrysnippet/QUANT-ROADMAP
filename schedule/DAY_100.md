## Block 2 — DSA · The Ultimate Decision Tree
1. **Connectivity / components / reachability?** → BFS / DFS / DSU.
2. **Shortest path?** unweighted → BFS; weighted → Dijkstra; negative edges → Bellman-Ford.
3. **Best / max / min / count with future consequences?** → DP.
4. **Fast search in a sorted answer space?** → Binary Search (incl. binary search on answer).
5. **Dependencies / ordering?** → Topological sort / Graph DP.
6. **Local best choice, provably safe?** → Greedy.

---

## Block 3 · Pattern Recognition Championship (Pattern + Algorithm + Reason)
(A) cheapest route between cities → Dijkstra; (B) course-prerequisite ordering → Topological sort; (C) max-reward dependency chain → Graph DP; (D) budget allocation → Knapsack DP; (E) compare two documents → LCS; (F) maximum meetings → Greedy; (G) first valid answer in a sorted space → Binary Search.

---

## Block 4 — Mathematics (Complexity intuition)
Recognise instantly: Binary Search O(log n); BFS/DFS O(V+E); Dijkstra O((V+E) log V); LCS O(n·m); Knapsack O(N·W).

**Exercise:** rank slowest-growing → fastest-growing: O(log n), O(n), O(n log n), O(n²).

---

## Block 5 — Quant Thinking (Modelling before solving)

Strong candidates ask *what algorithm?*; elite candidates ask *what is this problem really?* Portfolio optimization *looks* like finance but is **optimization + constraints** → often Knapsack. Research dependencies *look* like planning but are **DAG + optimization** → often Graph DP.

**Hard puzzle:** is your roadmap (C++ → DSA → Probability → Statistics → ML → Quant Research) an array, a graph, DP, or a combination? Explain.

**Career connection:** strong quant researchers are **problem modelers**, not algorithm memorizers — real problems arrive as messy situations, not "LeetCode #327." Your job: *reality → abstraction → algorithm.*
