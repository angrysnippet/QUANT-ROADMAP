## Block 1 — The RL loop
An **agent** observes a **state**, takes an **action**, receives a **reward**, and lands in a new state — repeatedly, aiming to maximise **cumulative reward**:
```
state → action → reward + next state → …   (maximise Σ γᵗ · rewardₜ)
```
`γ` (discount factor) weights future vs immediate reward.

*Why it matters:* trading *is* sequential decision-making — your action (trade) changes the state (inventory, market impact) and the reward (P&L) is delayed and noisy.

---

## Block 2 — Exploration vs exploitation
Exploit (use the best known action) vs explore (try others to learn). Too much exploit → stuck in a local habit; too much explore → waste. Classic embodiment: the **multi-armed bandit** (which slot machine?) — the cleanest model of "allocate trials to uncertain options," directly relevant to capital allocation and A/B testing of signals.

**Concept check:** in live trading, why is excessive "exploration" dangerous? (Exploring = taking suboptimal real trades = real money lost; exploration is far cheaper in simulation.)

---

## Block 3 — Value & policy
- **Value function `V(s)` / `Q(s,a)`:** expected cumulative reward from a state (or state-action). The **Bellman equation** links them: `Q(s,a) = reward + γ·max_a' Q(s',a')` — *this is dynamic programming* (Day 136).
- **Policy `π(s)`:** the agent's action rule. RL methods either learn values (Q-learning) or directly optimise the policy (policy gradient).

---

## Block 4 — Mathematics (Bellman = DP, again)
The Bellman recursion is the same "optimal value = best immediate + discounted future optimal" you saw in shortest paths and DP. RL is **DP when the model/environment is unknown** — you *learn* transitions and rewards from experience instead of being handed them.

**Exercise:** why does a high `γ` (≈1) make the agent more "long-term"? (It weights distant rewards almost as much as immediate ones.)

---

## Block 5 — Quant Thinking (RL where it's real — and where it's hype)

**Genuine uses:**
- **Optimal execution:** how to slice a large order over time to minimise market impact (a textbook sequential-decision problem — Almgren–Chriss is the classic non-RL baseline).
- **Market-making:** quoting/inventory management as a control problem.
- **Dynamic hedging** under costs.

**Why RL for *alpha* is hard:** markets are **non-stationary, low-SNR, adversarial, and partially observable**; rewards are noisy and delayed; you can't safely "explore" with real capital; and a backtest "environment" is itself an overfit-prone model. RL alpha claims deserve heavy skepticism.

**Hard puzzle:** why is execution a *better* RL fit than return prediction? (Execution has a clear, immediate-ish reward (cost vs benchmark), a controllable action (trade rate), and structure that's more stationary than return-generating processes — the environment is closer to well-defined.)

**Career connection:** RL appears in execution/market-making teams. Showing you know **why it suits control problems but is treacherous for alpha** (non-stationarity, exploration cost, sim overfitting) is exactly the maturity firms want.

## Block 6 — Python (quant) · Bandit: explore vs exploit
```python
import numpy as np
rng = np.random.default_rng(0)
true = [0.02, 0.05, 0.01]           # unknown "edge" of 3 strategies
Q = [0,0,0]; n = [0,0,0]; eps = 0.1
for t in range(20000):
    a = rng.integers(3) if rng.random() < eps else int(np.argmax(Q))  # ε-greedy
    r = rng.normal(true[a], 1.0)
    n[a]+=1; Q[a]+=(r-Q[a])/n[a]    # incremental mean (online update)
print([round(q,3) for q in Q], "picks:", n)   # converges to best arm (#1)
```

## Quant Thinking Track — Decisions That Change the World
Supervised learning predicts a *fixed* world; RL acts in a world its actions *change* (market impact!). That feedback — your trade moves the price — is why execution is control, not prediction.
