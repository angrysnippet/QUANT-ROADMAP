## The big unification
Every problem can often be viewed as **state → possible actions → new state.** Maze: `(row, col)` + move → `(new_row, new_col)`. Flights: `(city)` + take-flight → `(new_city)`. Both are **state graphs.**

---

## Block 1 — BFS as state-space search
State = node; goal = minimum steps. BFS explores distance 0, 1, 2, … layer by layer.

**Deep insight:** BFS is *shortest path in an **unweighted** state space* — not merely a graph trick.

*Why it matters:* seeing one structure under many algorithms means you can adapt to unseen problems instead of memorising each.

---

## Block 2 — Dijkstra as state-space search
`A→B=10, A→C=2, C→B=1`: state = node; goal = minimum **cost.** BFS minimises *steps*; Dijkstra minimises *cost*, exploring the **lowest-cost state first.**

**Deep insight:** Dijkstra is *best-first exploration* of a state space.

---

## Block 3 — DP as state-space search
Fibonacci: state = `n`; transitions `n→n-1`, `n→n-2` — that's a **DAG.** DP computes the best value for every state.

**Huge insight:** DP is *state-space search with memoization* — not a separate universe.

---

## Block 4 — Mathematics (States + transitions)
- BFS: state = node, transition = edge.
- Dijkstra: state = node, transition = weighted edge.
- Grid DP: state = cell, transition = move.
- Graph DP: state = node, transition = dependency edge.

**Exercise:** for Climbing Stairs, what's the state and the transition? **Challenge:** why does defining states correctly solve half the problem?

## Block 5 — Quant Thinking (Everything is a state machine)

Your roadmap: state `(C++, arrays, functions)` + action "study DSA" → new state "stronger solver." Learning, businesses, research, trading, engineering all model as *state → action → new state.*

**Hard puzzle:** state `DSA=40, Probability=20, Python=30`, 3 study hours — what actions produce the best future state? Think transitions.

**Career connection:** reinforcement learning (state = market, action = trade, new state = updated portfolio) and portfolio management share this exact structure — the language changes, the structure stays.
