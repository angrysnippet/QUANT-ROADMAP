## Block 1 — Why depth?
Each layer transforms the previous layer's outputs, building **hierarchical features**: edges → shapes → objects (vision); characters → words → meaning (language). Depth lets the model *compose* simple patterns into complex ones with far fewer parameters than a shallow net would need.

*Why it matters:* depth shines when the signal is genuinely **compositional and abundant** — true for images/text/order-books, much less so for a handful of daily price features.

---

## Block 2 — Architectures (know the map)
- **CNN** — spatial/local structure (images; sometimes order-book "images").
- **RNN / LSTM / GRU** — sequences with memory (older time-series approach).
- **Transformer (attention)** — long-range dependencies; now dominant in NLP and increasingly sequence modelling. Attention = "which past elements matter for this one."

**Concept check:** which architecture for sentiment from news headlines? (Transformer/NLP — text sequences with long-range context.)

---

## Block 3 — Making deep nets train
Depth created problems (vanishing gradients) solved by: **ReLU** activations, **batch/layer normalization**, **residual connections** (skip connections), good initialization, and **Adam**. Plus heavy **regularization** (dropout, weight decay, early stopping, data augmentation).

---

## Block 4 — Mathematics (representation + the data hunger)
Deep models have enormous capacity → they need **enormous, high-SNR data** to avoid overfitting. The bias-variance optimum (Day 128) only moves toward deep models when data is plentiful and signal is strong.

**Exercise:** images have millions of high-signal pixels; daily equity returns have a few weak features over thousands of days. Which justifies a deep model, and why? (Images — high SNR + massive data; returns — neither.)

---

## Block 5 — Quant Thinking (a sober, current view)

Where deep learning is genuinely used in quant (2020s):
- **NLP / alternative data:** news, earnings calls, filings, social sentiment (Transformers).
- **Market microstructure:** deep models on **limit-order-book** data (high-frequency, high-volume → enough signal+data).
- **Execution / market-making:** sequence models for short-horizon dynamics.
- **Derivatives:** fast neural *approximations* of expensive pricers ("deep pricing/calibration").

Where it usually **disappoints**: low-frequency return forecasting from small feature sets — simple, regularized, economically-motivated models remain king.

**Hard puzzle:** a vendor sells a "deep learning alpha" with Sharpe 5 on daily equities. Reaction? (Extreme skepticism — almost certainly overfit/leaked/selection-biased; deep nets on low-SNR daily data don't yield Sharpe 5. Demand walk-forward, costs, capacity, and trial count.)

**Career connection:** sophistication is shown by *matching tool to problem*: deep learning for high-data/compositional tasks (NLP, LOB), simple models for noisy low-freq returns. Blanket "use deep learning" is a junior tell.

## Quant Thinking Track — Depth Where It's Deserved
Depth is representation power; representation power needs rich, abundant signal to be worth its variance cost. Use it where data is deep, not where it's merely fashionable.
