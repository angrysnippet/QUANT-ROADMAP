## Block 4 — Mathematics · the unifying engineering principles
The module's deep principles, distilled:
```
Mechanical sympathy:  understand the machine (cache, pipeline, branches) → real performance
Tail focus:           optimize/measure the TAIL (p99.9 latency) like tail RISK (CVaR) — worst case dominates
Structural correctness: encode invariants (RAII, risk-gate, lookahead-impossible) → bugs impossible by construction
Amdahl's Law:         overall speedup bounded by bottleneck fraction → profile, optimize the dominant cost
Research-prod parity: deployed system == validated system → the validation actually holds
Zero-cost abstraction: abstraction resolved at compile time → expressive AND fast
```
Three unifications tie it to the whole curriculum: (1) **tail-latency engineering mirrors tail-risk management** (worst case dominates — Days 215/216); (2) **structural correctness mirrors robust design** (encode the invariant, don't rely on discipline — Days 218/224); (3) **verify-don't-hope mirrors honest research** (measure, don't assume — Days 199/222).

## Block 6 — C++ (systems) · The integrated trading-system skeleton
```cpp
#include <vector>
#include <atomic>
#include <memory>
#include <chrono>
#include <iostream>

// integration of the module: event-driven, cache-friendly, RAII, zero-alloc, tested-in-spirit
struct MarketEvent { long ts; double price; };

class SignalEngine {                                  // the strategy — shared backtest & live (parity)
    std::vector<double> window_;                      // contiguous, cache-friendly (Day 222)
    const std::size_t W_ = 20;
public:
    SignalEngine() { window_.reserve(W_); }           // pre-allocate (Day 226: zero-alloc hot path)
    int on_event(const MarketEvent& e) {              // react to event — only PAST data (lookahead-safe)
        if (window_.size() == W_) window_.erase(window_.begin());
        window_.push_back(e.price);
        if (window_.size() < W_) return 0;
        double mean = 0; for (double p : window_) mean += p; mean /= W_;   // stable, SIMD-able (227)
        return e.price < mean*0.98 ? 1 : (e.price > mean*1.02 ? -1 : 0);   // signal
    }
};

int main() {
    SignalEngine engine;                              // RAII: cleans up automatically (Day 224)
    std::vector<MarketEvent> feed;                    // historical (backtest) or live events
    for (int i=0;i<10000;++i) feed.push_back({i, 100.0 + 5*std::sin(i*0.1)});

    // event loop — measure TAIL latency of the hot path (Day 226/229)
    std::vector<long> lat; int pos=0; double pnl=0;
    for (const auto& ev : feed) {
        auto t0 = std::chrono::high_resolution_clock::now();
        int sig = engine.on_event(ev);                // hot path: signal computation
        auto t1 = std::chrono::high_resolution_clock::now();
        lat.push_back(std::chrono::duration_cast<std::chrono::nanoseconds>(t1-t0).count());
        // (risk gate would go here — Day 228) then execute
        pos += sig*10; pnl -= sig*10*ev.price + std::abs(sig*10)*0.01;   // net of cost
    }
    std::sort(lat.begin(), lat.end());
    std::cout << "P&L (net): " << pnl + pos*feed.back().price << "\n";
    std::cout << "signal latency: p50=" << lat[lat.size()/2]
              << "ns p99.9=" << lat[size_t(0.999*lat.size())] << "ns (the TAIL matters)\n";
    std::cout << "→ same SignalEngine runs live (parity); cache-friendly, zero-alloc, tail-measured\n";
}
```
The integrated skeleton: an **event-driven** system with a **cache-friendly, pre-allocated (zero-alloc)** signal engine that's the *same code* for backtest and live (**parity**), **RAII**-managed, lookahead-safe, and with **tail-latency** measurement — the module's pieces assembled into a trustworthy, fast trading-system core.

**Linux:** `g++ -O3 -march=native -std=c++20 system.cpp -o system && taskset -c 2 ./system` (core-pinned); verify with `-fsanitize=address,undefined` (correctness) and `perf stat` (cache/branches) — the full *verify-don't-hope* discipline (Day 229).
