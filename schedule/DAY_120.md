## Block 1 — Variance & standard deviation
```
Var(X) = E[(X − μ)²] = E[X²] − (E[X])²       σ = √Var(X)
```
Variance is the average squared deviation from the mean; **σ** (std dev) is in the same units as `X`. Two bets with identical EV can have wildly different σ — and σ is *risk*.

**Example:** Bet A: ±₹1 fair. Bet B: ±₹1000 fair. Same EV (0), vastly different variance. You'd size them completely differently.

---

## Block 2 — Key identities
- `Var(aX + b) = a²·Var(X)` (shifting doesn't change spread; scaling squares).
- **Independent sum:** `Var(X + Y) = Var(X) + Var(Y)`.
- **General sum:** `Var(X + Y) = Var(X) + Var(Y) + 2·Cov(X, Y)`.
- `Cov(X, Y) = E[XY] − E[X]E[Y]`; **correlation** `ρ = Cov(X,Y)/(σ_X σ_Y) ∈ [−1, 1]`.

**Concept check:** variance of the sum of `n` i.i.d. variables each with variance `σ²`? (`nσ²`; std `σ√n` — the **√n law**, why averages stabilise.)

---

## Block 3 — Why this is the heart of portfolio risk
Portfolio return `R = Σ wᵢ Rᵢ`. `E[R] = Σ wᵢ E[Rᵢ]` (linearity), but
```
Var(R) = Σᵢ Σⱼ wᵢ wⱼ Cov(Rᵢ, Rⱼ)
```
**Diversification** = adding assets with low/negative covariance lowers portfolio variance *without* lowering expected return proportionally. This single formula is the core of Markowitz portfolio theory (later module).

---

## Block 4 — Mathematics (the √n law & Sharpe)
Averaging `n` i.i.d. draws: mean variance `σ²/n`, std `σ/√n` → uncertainty shrinks like `1/√n`. **Sharpe ratio** (return per unit risk):
```
Sharpe = (E[R] − r_f) / σ_R
```
**Exercise:** strategy A returns 8% with σ=4%; B returns 12% with σ=12% (r_f=0). Which has the better Sharpe? (A: 2.0 vs B: 1.0 → A, despite lower return.) **This is how quants actually compare strategies** — risk-adjusted, not raw return.

---

## Block 5 — Quant Thinking (Risk is priced)

- **EV says "play"; variance says "how much."** The **Kelly criterion** sizes bets using both: bet fraction `f* ≈ edge/odds`; over-betting a positive-EV game still ruins you (yesterday's puzzle).
- **Fat tails:** real return distributions are **not** Normal — they have far more extreme moves (a "6-sigma" day happens far more than Normal predicts). Models that assume Normality (and ignore tail risk) blow up. *This humility separates surviving quants from blown-up ones.*
- **Drawdown** (peak-to-trough loss) often matters more than variance to investors and risk managers.

**Hard puzzle:** two strategies, same annual return 10%. A: σ=8%, max drawdown 12%. B: σ=8%, max drawdown 40% (rare but brutal). Same variance-ish, very different *survivability* — why can σ hide tail risk? (Variance under-weights rare extremes; you also need tail measures like **VaR/CVaR**, later module.)

**Career connection:** "compare these two strategies" in an interview → talk **Sharpe, drawdown, and tail risk**, never raw return alone. This instantly signals real quant maturity.

## Block 6 — Python (quant) · Risk metrics on returns
```python
import numpy as np
rng = np.random.default_rng(0)
ret = rng.normal(0.0005, 0.01, 252)      # ~1 trading year of daily returns
ann_ret  = ret.mean() * 252
ann_vol  = ret.std(ddof=1) * np.sqrt(252) # √n scaling to annualise
sharpe   = ann_ret / ann_vol
print(round(ann_ret,3), round(ann_vol,3), round(sharpe,2))
```
Bonus: compute max drawdown from a cumulative-return path; compare a Normal sample vs a heavy-tailed (Student-t) sample and see how many "5σ" days each produces.
