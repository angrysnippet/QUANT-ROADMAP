## Block 1 — The Markov property
A process is **Markov** if the next state depends only on the *current* state, not the full history:
```
P(Sₜ₊₁ | Sₜ, Sₜ₋₁, …, S₀) = P(Sₜ₊₁ | Sₜ)
```
"The present screens off the past." This is a modelling *assumption* — powerful when roughly true, dangerous when not.

*Why it matters:* it makes otherwise intractable dynamics solvable, and underlies random walks, MDPs (tomorrow), and regime-switching models.

---

## Block 2 — The transition matrix
For `n` states, `P` is an `n×n` matrix where `P[i][j] = P(next = j | now = i)`; **rows sum to 1.** The distribution after `k` steps is `π₀ · Pᵏ`.

**Example — market regimes** {Bull, Bear}:
```
P = [[0.9, 0.1],     # Bull → stays Bull 90%
     [0.2, 0.8]]      # Bear → stays Bear 80%
```
Persistent regimes (high diagonal) — markets cluster, exactly as observed.

---

## Block 3 — Stationary distribution
The long-run fraction of time in each state, `π`, satisfies `π = π·P` (a left eigenvector with eigenvalue 1). For the regime example, solving gives `π ≈ (Bull 0.67, Bear 0.33)`.

**Concept check:** what does the stationary distribution tell a risk manager? (The unconditional, long-run probability of being in each regime — base rates for stress and capital.)

---

## Block 4 — Mathematics (compute it)
`π·P = π` with `Σπ = 1`. For the 2-state chain: `π_Bull·0.1 = π_Bear·0.2` → `π_Bull = 2·π_Bear` → `(2/3, 1/3)`. **Exercise:** verify by powering `P` many times — every row converges to `π`. **Challenge:** when does a chain *fail* to have a unique stationary distribution? (Reducible or periodic chains — no mixing.)

## Block 5 — Quant Thinking (Markov in markets)

- **Regime-switching models** (Hamilton): returns/volatility differ by regime; transitions are Markov. Used for risk and allocation.
- **Credit migration matrices:** rating agencies publish `P` for AAA→AA→…→Default — a Markov chain priced into bonds.
- **The random walk** (tomorrow) is a Markov process; the **efficient-market** idea is essentially "prices are (near) Markov — the past doesn't help predict the future."

**Hard puzzle:** a strategy works great in "calm" regimes and blows up in "crisis." Why is ignoring regime *transitions* dangerous, even if crises are rare? (The transition probability into crisis, times the catastrophic loss, can dominate expected P&L — and crises cluster, so the chain spends correlated time there. Stationary base rates understate path risk.)

**Career connection:** "how would you model market regimes?" → Markov/regime-switching, transition matrix, stationary distribution. Also: credit-migration matrices are a standard fixed-income interview topic.

## Block 6 — Python (quant) · Simulate a regime chain
```python
import numpy as np
rng = np.random.default_rng(0)
P = np.array([[0.9,0.1],[0.2,0.8]])
state, counts = 0, np.zeros(2)
for _ in range(100_000):
    counts[state]+=1
    state = rng.choice(2, p=P[state])
print(counts/counts.sum())          # ≈ [0.667, 0.333] stationary dist
# Verify via eigenvector:
w,v = np.linalg.eig(P.T); pi = np.real(v[:,np.argmin(abs(w-1))]); print(pi/pi.sum())
```

**Linux:** `python -c "import numpy as np; print(np.linalg.matrix_power(...))"` for quick `Pᵏ`.
