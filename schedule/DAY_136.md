## Block 1 — The Principle of Optimality
*An optimal policy has the property that, whatever the initial state and decision, the remaining decisions must be optimal from the resulting state.* (Bellman, 1957.) In short: **the tail of an optimal plan is itself optimal** — so you can solve recursively.

*Why it matters:* this is *why* DP works, and it's the engine of dynamic hedging, optimal execution, and pricing American options.

---

## Block 2 — The Bellman equation
For a sequential decision problem with discount `γ`:
```
V*(s) = max_a [ reward(s,a) + γ · Σ P(s'|s,a) · V*(s') ]
```
The optimal value of a state = best immediate reward + discounted expected optimal value of where you land. **Value iteration** just applies this update until it converges.

**Concept check:** with no uncertainty and `γ=1`, what does this reduce to? (Longest/shortest path on a DAG — Day 111.)

---

## Block 3 — Backward induction (the practical method)
When the horizon is finite, solve **backward from the end**: the value at the last step is known; step back computing each state's optimal value from the already-solved future. This is exactly how you price an American option on a tree and how Almgren–Chriss-style execution schedules are derived.

**Exercise:** a 3-step game; the final payoff is known at each end state. How do you find the best first move? (Compute values at step 3, then 2, then 1 — backward induction.)

---

## Block 4 — Mathematics (deterministic vs stochastic)
- **Deterministic:** `V(s) = max_a [reward + V(s')]` (shortest path, knapsack).
- **Stochastic:** the `Σ P(s'|s,a) V(s')` term averages over random next states — the bridge to MDPs (tomorrow) and to risk-neutral pricing (expectation of discounted payoff).

**Challenge:** why does discounting (`γ<1`) guarantee value iteration converges? (It's a contraction mapping — each update shrinks errors by factor γ.)

---

## Block 5 — Quant Thinking (Bellman is everywhere in finance)

- **American option pricing:** at each node, `value = max(exercise now, E[discounted continuation])` — a Bellman equation solved by backward induction.
- **Optimal execution:** minimise `E[cost] + λ·risk` of liquidating over time — a stochastic control problem.
- **Dynamic portfolio choice (Merton):** how much to hold over time = Bellman optimization of expected utility.

**Hard puzzle:** why can a holder of an American put *optimally exercise early*, while a European one can't? (American = a decision at every node → Bellman lets `exercise now` beat `continuation value` when deep in-the-money; European has a single terminal decision.)

**Career connection:** "how would you price an American option / schedule a large trade?" → *backward induction / Bellman.* Recognising sequential decisions as Bellman problems is core quant maturity.

## Block 6 — Python (quant) · Value iteration on a tiny MDP-ish grid
```python
import numpy as np
# Backward induction: a 3-step binomial value (continuation vs exercise) sketch
S0, u, d, K, r = 100, 1.1, 0.9, 100, 0.0
# terminal stock prices after 3 steps
import itertools
def price_tree(steps=3, p=0.5, gamma=1.0):
    # values at maturity = put payoff
    vals = [max(K - S0*u**i*d**(steps-i), 0) for i in range(steps+1)]
    for step in range(steps-1, -1, -1):       # backward induction
        new=[]
        for i in range(step+1):
            S = S0*u**i*d**(step-i)
            cont = gamma*(p*vals[i+1] + (1-p)*vals[i])
            new.append(max(K - S, 0 if False else 0) if False else max(max(K-S,0), cont))  # American put
        vals=new
    return vals[0]
print(round(price_tree(),3))
```
(The point is the *backward sweep*, not the toy numbers.)
