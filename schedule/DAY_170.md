## Block 1 — The one idea: discount cash flows on a curve
Every instrument in this module was the same operation with more realism layered on:
- **Bond** = cash flows discounted on the curve (Day 161).
- **Duration/convexity** = the first/second derivatives of that price w.r.t. yield (162–163).
- **The curve** = the full set of discount factors, with forwards by no-arbitrage (164–165).
- **Curve risk** = the price's sensitivity to *each mode* of the curve (166).
- **Credit** = discount at curve + spread, where the spread prices default (a short put, 167).
- **Rate models** = make the curve itself stochastic, for consistent pricing (168).
- **Swaps/caps/swaptions** = derivatives discounted and optioned off the curve (169).

Remember one thing from Phase 9: **price = Σ (cash flow × discount factor); everything else is *which* cash flows and *what* curve.**

---

## Block 2 — The fixed-income pipeline (your mental flowchart)
```
1. Identify the cash flows (fixed? floating? contingent on default?).
2. Build the discount curve (bootstrap from liquid bonds/swaps).
3. Price = Σ CF × DF.   For options on rates → Black / a calibrated model.
4. Risk: compute duration, DV01, convexity, key-rate durations.
5. Credit-risky? Add a spread; model PD×LGD (reduced-form λ or Merton).
6. Hedge: match DV01 / key-rate durations; neutralize level/slope/curvature.
7. Quote in yield/spread/swap-rate, not dollars.
```
This routes essentially any rates instrument you'll meet.

## Block 3 — The mental models, distilled
- **Price–yield is inverse and convex** (161/163): rates up → prices down, less than linear.
- **Duration = the rate delta; DV01 = dollar risk; convexity = the gift** (162–163).
- **The curve is four views of one object** — yield/spot/forward/DF (164).
- **Forwards are biased forecasts** = expectation + term premium → carry (165).
- **~3 factors (level/slope/curvature) drive the curve** (166).
- **Risky debt = risk-free debt − a put on the firm** (Merton, 167).
- **Rates mean-revert; calibrate, don't forecast** (168).
- **Swap = strip of FRAs; cap = strip of caplets; same option toolkit** (169).

## Block 4 — Mathematics · the equations worth knowing cold
```
Bond price:     P = Σ CF_t · DF(t),     DF(t)=e^{−z(t)t}
Duration:       ΔP/P ≈ −ModDur·Δy + ½·Convexity·Δy²
DV01:           ModDur · P · 1e-4
Forward:        (1+z₂)^{t₂} = (1+z₁)^{t₁}(1+f)^{t₂−t₁}
Credit spread:  Spread ≈ PD × LGD (+ premium);  risky debt = RF debt − Put
Vasicek:        dr = a(b−r)dt + σ dW
Swap rate:      (1 − DF_N) / Σ DF_i τ_i
```
If you can price a bond, expand its risk to second order, derive a forward, and write `risky debt = risk-free − put`, you can reconstruct the module under pressure.

## Block 5 — Quant Thinking · interview rehearsal (say these out loud)
1. *Rates rise — what happens to bond prices, and which bonds worst?* → fall (inverse price–yield); worst = long-maturity, low-coupon = highest duration (161–162).
2. *You've duration-hedged — what's left?* → curve risk: slope & curvature; hedge with key-rate durations / the 3 PCA factors (166).
3. *Why is convexity good for you?* → it's positive gamma for free: more upside when rates fall, less downside when they rise; only matters for big moves (163).
4. *What's a credit spread, and what's the Merton insight?* → compensation for PD×LGD + premium; equity = call on assets, risky debt = risk-free − put (167).
5. *Why not model rates with GBM?* → rates mean-revert and must keep the curve consistent; use Vasicek/CIR/Hull–White, and calibrate rather than forecast (168).
6. *Hedge floating debt: swap or cap?* → swap is free/symmetric (gives up the downside benefit); cap costs premium but is convex/asymmetric (keeps it) (169).

**Hard puzzle (capstone):** a pension owes \$100M in 10 years and holds \$60M today. Design a strategy and name every risk. (**Immunize**: invest the \$60M in bonds with **duration ≈ 10** to match the liability horizon (166), so small parallel rate moves hit assets and the liability's PV equally. Risks to manage: **(1) reinvestment/level** — handled by duration matching to first order; **(2) curve risk** — match *key-rate* durations, not just one number, or a twist breaks the hedge; **(3) convexity mismatch** — the liability is a single long cash flow (bullet, low convexity); match convexity or a large move leaves a gap; **(4) credit risk** — if you reach for yield in corporates you add PD×LGD and a short-put profile (167); **(5) the funding gap itself** — \$60M growing to \$100M needs ~5.2%/yr, so you're *also* implicitly taking return risk, not just hedging. This one question touches discounting, duration, key-rate hedging, convexity, and credit — the whole module.)

**Career connection:** Phase 9 *is* the rates/ALM interview. A candidate who prices a bond, expands risk through convexity, hedges the curve with key-rate durations, prices credit via Merton, and structures an immunization with its full risk list is demonstrably desk-ready for fixed income, ALM, or rates trading.

## Block 6 — Python (quant) · A unified mini fixed-income toolkit
```python
import numpy as np
from scipy.optimize import brentq

class FI:
    def __init__(self, mats, spot):
        self.mats, self.spot = np.array(mats), np.array(spot)
    def DF(self, t): return (1+np.interp(t,self.mats,self.spot))**(-t)
    def bond(self, cpn, n, face=100, freq=2):
        t=np.arange(1,n*freq+1)/freq; c=face*cpn/freq
        cf=np.full(len(t),c); cf[-1]+=face
        return float((cf*self.DF(t)).sum())
    def dv01(self, cpn, n, face=100):
        h=1e-4; up=FI(self.mats,self.spot+h).bond(cpn,n,face)
        dn=FI(self.mats,self.spot-h).bond(cpn,n,face)
        return (dn-up)/2/h*1e-4
    def swap_rate(self, n):
        t=np.arange(1,n+1); df=self.DF(t); return float((1-df[-1])/df.sum())

c = FI([1,2,3,5,10],[0.030,0.034,0.037,0.040,0.042])
print(f"5y 4% bond price = {c.bond(0.04,5):.3f}")
print(f"5y bond DV01     = {c.dv01(0.04,5):.4f} per $100")
print(f"5y par swap rate = {c.swap_rate(5)*100:.3f}%")
```
Curve in → bond price, DV01, and swap rate out — the whole module in one small class you fully understand.
