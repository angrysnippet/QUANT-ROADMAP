## Block 1 — The five components
An MDP is `(S, A, P, R, γ)`:
- **S** states, **A** actions,
- **P(s'|s,a)** transition probabilities (Markov),
- **R(s,a)** rewards, **γ** discount.

A **policy `π(s)`** maps states → actions. Goal: find `π*` maximising expected discounted reward.

*Why it matters:* execution, market-making, hedging, and dynamic allocation are all MDPs — once you see the structure, the solution methods are standard.

---

## Block 2 — Value functions & Bellman optimality
```
Vπ(s)  = E[ Σ γᵗ R | start s, follow π ]
V*(s)  = max_a [ R(s,a) + γ Σ P(s'|s,a) V*(s') ]      (Bellman optimality)
π*(s)  = argmax_a [ R(s,a) + γ Σ P(s'|s,a) V*(s') ]
```
Same Bellman equation as Day 136, now with a choice of action at each state.

---

## Block 3 — Solving an MDP
- **Value iteration:** repeatedly apply the Bellman optimality update until `V` converges, then read off the greedy policy.
- **Policy iteration:** alternate *evaluate current policy* and *improve it greedily* — often fewer iterations.
- **When P and R are unknown → Reinforcement Learning** (Day 135): learn them from experience.

**Concept check:** DP vs RL in one line? (DP = MDP with a *known* model; RL = MDP with an *unknown* model, learned from data.)

---

## Block 4 — Mathematics (contraction → convergence)
The Bellman optimality operator is a **γ-contraction**: each iteration brings `V` closer to `V*` by factor γ, so value iteration converges geometrically. **Exercise:** with γ=0.9, roughly how many iterations to cut error by 1000×? (`0.9ⁿ < 0.001` → n ≈ 66.) **Challenge:** why does γ=1 with infinite horizons risk non-convergence? (Rewards can sum to infinity; need γ<1 or an absorbing/terminal structure.)

---

## Block 5 — Quant Thinking (the desk problems that *are* MDPs)

- **Optimal execution:** state = (inventory left, time left, market state); actions = trade rate; reward = − (cost + risk). Solve for a liquidation schedule.
- **Market-making:** state = (inventory, spread, queue); actions = quotes; reward = spread captured − inventory risk.
- **Dynamic hedging / Merton portfolio:** state = (wealth, prices); action = allocation; reward = utility of wealth.

**Hard puzzle:** in optimal execution, why does the optimal policy usually **trade faster when volatility is high**? (Higher vol → more risk from holding unexecuted inventory → the Bellman tradeoff shifts toward paying more impact now to cut risk. The MDP makes this precise.)

**Career connection:** "frame optimal execution as an MDP" or "what's the state/action/reward for a market-maker?" are real execution-desk interview questions. The MDP vocabulary signals you can model control problems, not just predict.

## Block 6 — Python (quant) · Value iteration
```python
import numpy as np
# Tiny MDP: 3 states, 2 actions; arbitrary P, R for illustration
S, A = 3, 2; gamma=0.9
P = np.random.default_rng(0).dirichlet(np.ones(S), size=(S,A))  # P[s,a] -> dist over s'
R = np.array([[0.0,1.0],[ -1.0,0.5],[2.0,0.0]])
V = np.zeros(S)
for _ in range(200):
    Q = R + gamma*(P @ V)              # Q[s,a]
    V = Q.max(axis=1)                  # Bellman optimality update
policy = Q.argmax(axis=1)
print(np.round(V,3), policy)
```
