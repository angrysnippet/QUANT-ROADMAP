## Block 1 — Definition
For a discrete random variable: `E[X] = Σ xᵢ · P(xᵢ)`. For continuous: `E[X] = ∫ x f(x) dx`. It's the **probability-weighted average** — the long-run average outcome.

- Fair die: `E[X] = (1+2+3+4+5+6)/6 = 3.5`.
- Bet: win ₹10 w.p. 0.4, lose ₹5 w.p. 0.6 → `E = 0.4·10 − 0.6·5 = +1` → favourable.

*Why it matters:* "what's the fair value?" = "what's the expected value?" — the question under every quote.

---

## Block 2 — Linearity of expectation (the superpower)
`E[X + Y] = E[X] + E[Y]` — **always, even if X and Y are dependent.** This turns many "hard" problems trivial.

**Classic:** expected number of fixed points when n people get hats back at random? Each person matches w.p. `1/n`; by linearity `E = n·(1/n) = 1` — regardless of dependence. No messy casework.

**Concept check:** expected number of heads in 100 tosses of a biased coin (p=0.3)? `100·0.3 = 30` — linearity again.

## Block 3 — Expectation of classic games (interview gold)
- **Coupon collector:** expected draws to collect all `n` coupons `= n·Hₙ ≈ n ln n` (sum of geometric waits `n/n + n/(n−1) + … + n/1`).
- **Expected tosses for the first Head:** Geometric → `1/p`.
- **Expected tosses for "HH":** set up states (`E = ...`) → **6**; for "HT" → **4**. (They differ — a famous trap; HH must "restart" on a tail-after-head.)

**Task:** derive expected rolls of a fair die until the first 6 using the one-step equation `E = 1 + (5/6)E` → `E = 6`.

---

## Block 4 — Conditional expectation & the tower property
`E[X] = E[ E[X | Y] ]` — average the conditional averages. This is how you handle "it depends on what happens first."

**Example:** a game pays the roll of a die; you may *reroll once* if you wish. Optimal threshold: reroll if the first roll < `E[reroll] = 3.5`, i.e. reroll on 1–3. Value `= (E of max(first, …))` → `E = (4+5+6)/6 + (3.5)·(3/6) = 2.5 + 1.75 = 4.25`.

---

## Block 5 — Quant Thinking (EV is the job)

- **Pricing:** a derivative's fair value ≈ `E[ payoff ]` under the right (risk-neutral) measure — you'll formalise this in the pricing module.
- **Edge:** a strategy is worth trading only if `E[return per trade] > costs` (fees + slippage). Positive EV is necessary; it is **not** sufficient (variance can ruin you — tomorrow).
- **Linearity at portfolio scale:** `E[portfolio] = Σ E[positions]` exactly — even with correlations. (Variance is the part that needs the covariances.)

**Hard puzzle (the EV trap):** a game: 50% double your money, 50% lose 40%. Positive EV per play (`0.5·2 + 0.5·0.6 = 1.30`× expected). Should you bet your whole bankroll repeatedly? **No** — the *median* outcome decays (geometric vs arithmetic mean). This is the seed of the **Kelly criterion** and why variance matters.

**Career connection:** "what's the fair price of this bet/game?" and "what's the expected number of …?" are the single most common quant-interview question types — linearity and one-step state equations crack most of them.

## Block 6 — Python (quant) · Monte-Carlo EV
```python
import numpy as np
rng = np.random.default_rng(0)
# Expected tosses to first head
def first_head():
    n=1
    while rng.random() >= 0.5: n+=1
    return n
print(np.mean([first_head() for _ in range(200_000)]))   # ≈ 2.0
```
Bonus: simulate the "HH vs HT" waiting times and confirm 6 vs 4.
