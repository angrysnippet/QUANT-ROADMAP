## Block 1 — The universal principle
*Compute once, reuse many times.* You've seen it as DP memoization, hash maps, and rolling state. The same idea powers production quant infrastructure.

*Why it matters:* most real speedups come from **not repeating work**, not from a cleverer inner loop.

---

## Block 2 — Reuse in real quant systems
- **Backtest caches:** precompute indicators (moving averages, returns) once; reuse across strategy variants instead of recomputing per run.
- **Pricing memoization:** Monte-Carlo and lattice pricers cache intermediate node values; calibration reuses prior solutions as warm starts.
- **Market-data layers:** `symbol → latest quote` hash maps give O(1) lookups; feature stores cache computed signals.

**Thinking question:** what's the cost of a *stale* cached price vs a slightly slower fresh one? (In trading, stale can be far worse — correctness beats speed.)

---

## Block 3 — Python (quant) · Memoized vs recomputed
Compare two ways to compute rolling returns over a 10⁶-point price series: (a) recompute each window, (b) cache cumulative sums. Time both. The cached version should be dramatically faster — same big-O, different constant, plus reuse.

```python
import numpy as np
prices = np.random.lognormal(mean=0, sigma=0.01, size=1_000_000).cumprod()
log_ret = np.diff(np.log(prices))      # vectorised, computed once, reused everywhere
```

## Block 5 — Quant Thinking (Reuse = edge preservation)
A research team that caches cleaned data, computed features, and prior model fits iterates **10× faster** — and iteration speed is research edge. The lesson generalises: protect scarce resources (time, compute) by reusing the abundant ones (memory, storage).

**Career connection:** "how would you speed up this backtest?" → the senior answer is usually *precompute and reuse*, plus vectorize, not "rewrite the loop."
