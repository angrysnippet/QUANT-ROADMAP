## Block 1 — Weak (covariance) stationarity, precisely
A series is **weakly stationary** if for all `t`:
1. `E[xₜ] = μ` (constant mean),
2. `Var(xₜ) = σ²` (constant variance),
3. `Cov(xₜ, xₜ₊ₖ)` depends only on the lag `k`, not on `t`.

That's it — the *distributional fingerprint* doesn't drift with calendar time. Stationarity is what lets you estimate one set of parameters from history and trust them tomorrow.

*Why it matters:* if the mean or variance is itself moving, "the average return" is a meaningless number — you're averaging apples from different worlds.

---

## Block 2 — The random walk has a *unit root*
Write an AR(1): `xₜ = φ·xₜ₋₁ + εₜ`.
- `|φ| < 1` → **stationary**, mean-reverting, shocks decay geometrically.
- `φ = 1` → **unit root** = random walk: shocks are *permanent*, variance `= tσ²` grows without bound. **Non-stationary.**

Prices behave like `φ ≈ 1`. **Differencing** once (`Δxₜ = xₜ − xₜ₋₁`, i.e. taking returns) removes the unit root → returns are (roughly) stationary. A series needing `d` differences to become stationary is "integrated of order `d`," `I(d)`. Prices ≈ `I(1)`, returns ≈ `I(0)`.

---

## Block 3 — Testing: Augmented Dickey–Fuller (ADF)
ADF tests `H₀: unit root (non-stationary)` vs `H₁: stationary`. It regresses `Δxₜ` on `xₜ₋₁` (plus lagged differences) and checks if the `xₜ₋₁` coefficient is significantly negative (pulling back to a mean).
- **Reject H₀ (small p-value)** → stationary, mean-reverting → you can model the level.
- **Fail to reject** → unit root → difference it first.

**Concept check:** ADF on a price series typically *fails* to reject (unit root); on its returns it *rejects* (stationary). That's the empirical signature of a random walk.

---

## Block 4 — Spurious regression: the danger stationarity guards against
Regress one independent random walk on another. You'll routinely get a *high R² and a "significant" t-stat* — pure garbage, because two trending non-stationary series will look related by chance (Granger–Newbold, 1974). This is why you **never regress price on price**; regress returns on returns, or use **cointegration** (Day 147+ pairs-trading preview) when the *combination* is stationary even though each leg isn't.

**Hard puzzle:** your "model" regresses S&P price on a money-supply level and gets R²=0.98. Why is this almost certainly spurious, and what's the fix? (Both are `I(1)` trending series; high R² is the Granger–Newbold artifact. Fix: difference both / test for cointegration / use an error-correction model.)

---

## Block 5 — Quant Thinking · stationarity is a *modelling contract*
Stationarity isn't a property you passively check — it's a contract you *engineer*: differencing, taking returns, detrending, deflating, demeaning rolling windows. Markets are non-stationary in the deep sense (regimes change — Day 137), so even "stationary" returns have drifting variance. The pragmatic stance: **transform to approximate stationarity, then keep validating that the contract still holds out-of-sample.**

**Career connection:** "you have two correlated price series — how do you trade them?" The senior answer is *cointegration*, not correlation: test whether `y − βx` is stationary (ADF on the spread), trade the mean-reverting residual. Spotting the spurious-regression trap is a classic screening question.

## Block 6 — Python (quant) · ADF on price vs returns, and a spurious regression demo
```python
import numpy as np, pandas as pd
from statsmodels.tsa.stattools import adfuller
import statsmodels.api as sm
rng = np.random.default_rng(1)

price = 100 + np.cumsum(rng.standard_normal(1000))      # random walk (I(1))
ret   = np.diff(np.log(price))                          # returns (I(0))

print("ADF price  p-value:", round(adfuller(price)[1], 3))   # large -> non-stationary
print("ADF return p-value:", round(adfuller(ret)[1], 3))     # small -> stationary

# spurious regression: two INDEPENDENT random walks
a = np.cumsum(rng.standard_normal(1000))
b = np.cumsum(rng.standard_normal(1000))
m = sm.OLS(a, sm.add_constant(b)).fit()
print("spurious R²:", round(m.rsquared, 2), " |t| on b:", round(abs(m.tvalues[1]), 1))
```
You'll see the price ADF p-value large, the return one tiny, and the "independent" regression showing an absurd R² and t-stat — the lesson burned in by experiment.

**Linux:** `pip install statsmodels`; pipe results to a log and diff across runs to confirm the pattern is robust to the seed.

## Quant Thinking Track — Abstraction
"Stationary vs not" abstracts a messy series into a yes/no contract that determines *which entire toolbox* applies. One bit of information (`I(0)` or `I(1)`?) routes you to completely different methods. Good abstractions collapse complexity into decisions.
