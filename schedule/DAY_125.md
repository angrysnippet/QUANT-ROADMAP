## Block 1 — The framework
- **Null hypothesis `H₀`:** "no effect" (e.g. strategy's true mean return = 0).
- **Alternative `H₁`:** "there is an effect."
- **p-value:** probability of seeing data this extreme *if `H₀` were true.* Small p → data is surprising under "no effect."
- Reject `H₀` if `p < α` (commonly 0.05).

*Why it matters:* "is this edge real?" is a hypothesis test — and α=0.05 means **1 in 20 false alarms**, which becomes catastrophic when you test many ideas.

---

## Block 2 — The t-test (for a mean return)
```
t = (x̄ − μ₀) / (s/√n)        # μ₀ = 0 for "no edge"
```
Note `t = mean/SE`, and recall `t ≈ Sharpe·√n` — **the t-statistic and the Sharpe ratio are the same idea.** `|t| > ~2` ↔ significant at ~5%.

**Concept check:** a strategy, 252 days, daily mean 0.04%, daily sd 1% → `t = 0.04 / (1/√252) = 0.04·15.87 ≈ 0.63`. Significant? (No — nowhere near 2. One year can't confirm a small edge.)

---

## Block 3 — Type I vs Type II errors
- **Type I (false positive):** trade a strategy that's actually worthless. Rate = α.
- **Type II (false negative):** discard a real edge. Rate = β; **power = 1 − β** (Day 126).

In quant, a **Type I error costs money** (you deploy noise); a Type II costs **opportunity.** You tune α/β to your cost asymmetry.

---

## Block 4 — The multiple-testing problem (the big one)
Test 1 idea at α=0.05 → 5% false-positive chance. Test **100 independent** worthless ideas → expected `100·0.05 = 5` "significant" winners by **pure luck**, and `P(at least one) = 1 − 0.95¹⁰⁰ ≈ 99.4%`. **You are almost guaranteed to find fake alpha.**

Corrections:
- **Bonferroni:** require `p < α/m` for `m` tests (conservative).
- **False Discovery Rate (Benjamini–Hochberg):** control the *fraction* of discoveries that are false.
- **Deflated Sharpe ratio** (López de Prado): discount a backtest's Sharpe for the number of trials run.

**Exercise:** you tried 200 strategy variants and kept the best (Sharpe 2). Why is its naive p-value meaningless? (Selection over 200 trials — the max of many noisy estimates is large by construction; you must deflate.)

---

## Block 5 — Quant Thinking (Backtest overfitting = multiple testing)

- Every parameter you tune, every variant you try, is an implicit test. Tune enough knobs and a random series "predicts" returns.
- **Report the number of trials.** A Sharpe of 3 found after 1 attempt ≫ a Sharpe of 3 found after 10,000 attempts.
- **The honest workflow:** hypothesis → lock the test → run it **once** on held-out data. Re-using test data degrades it into training data.

**Hard puzzle:** two researchers both present a Sharpe-2 strategy. A tested 1 economically-motivated idea; B grid-searched 5,000 combinations and showed you the best. Same Sharpe — whose is more likely real, quantitatively? (A's, by a huge factor: B's "best of 5,000" is an extreme order statistic of noise. Ties straight back to Day 122's priors.)

**Career connection:** "how do you avoid overfitting in research?" → talk **train/validation/test discipline, deflated Sharpe, FDR control, and counting your trials.** This is precisely how top funds gate live capital.

## Block 6 — Python (quant) · Multiple testing in action
```python
import numpy as np
rng = np.random.default_rng(0)
# 1000 worthless signals vs random returns: how many look "significant"?
ret = rng.normal(0, 1, 252)
hits = 0
for _ in range(1000):
    sig = rng.normal(0, 1, 252)
    t = np.corrcoef(sig[:-1], ret[1:])[0,1] * np.sqrt(250)   # rough t
    if abs(t) > 1.96: hits += 1
print(hits)            # ≈ 50 false positives from pure noise
```
Bonus: apply a Bonferroni threshold (`α/1000`) and watch the false positives collapse.

**Linux:** `seq 1000 | wc -l` mindset — count your trials; if you don't track how many tests you ran, your p-values are fiction.
