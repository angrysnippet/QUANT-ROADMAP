## Why today matters
Given *a weighted graph + a budget + a reward function*, instantly answering "Dijkstra" is dangerous. Strong thinkers first ask: **what exactly is the problem?**

---

## Block 1 — The Universal Solver Framework
Never start with code — start with questions:
1. **Objects** — array? string? graph? grid? intervals? tree?
2. **Objective** — max? min? count? existence? shortest?
3. **Constraints** — budget? capacity? dependencies? distance? time?
4. **Patterns** — optimization? connectivity? matching? scheduling? search?

*Only then* choose an algorithm.

---

## Block 2 — DSA · Multi-Pattern Thinking
Hard problems often combine patterns:
- *"Minimum capacity to complete work"* = optimization + feasibility → **Binary Search + Greedy.**
- *"Maximum reward through dependencies"* = graph + optimization → **Graph DP.**

**Exercise:** cities + roads + minimum travel cost → object **Graph**, objective **Minimum**, algorithm **Dijkstra.**

---

## Block 3 — DSA · The Four Common Mistakes
1. **Keyword-matching only** — "maximum" → reflexively DP. Wrong; need full context.
2. **Ignoring constraints** — `n = 10⁵` kills quadratic solutions.
3. **Jumping to code** — strong solvers model first.
4. **Forgetting simpler solutions** — sometimes sorting beats DP.

---

## Block 4 — Mathematics (Complexity-based thinking)
Estimate before coding: `n=10` anything works; `n=10³` O(n²) is fine; `n=10⁵` need O(n log n); `n=10⁶` need near-linear.

**Exercise:** for `n=100000`, is O(n²) realistic? Why not? (10¹⁰ operations — far too slow.)

---

## Block 5 — Quant Thinking (Model before method)

*100 stocks + limited capital + expected returns* looks like finance, but is *items / value / weight / capacity* → **Knapsack.** A learning roadmap is *dependencies + rewards* → **Graph DP.**

**Deep insight:** expertise flows *reality → abstraction → algorithm*, not *algorithm → reality.*

**Career connection:** this separates a good student from a strong researcher — researchers ask *what is the underlying structure?*, not *what chapter is this from?*
